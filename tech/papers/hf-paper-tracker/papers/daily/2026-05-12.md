### [T3] PACEvolve++: Improving Test-time Learning for Evolutionary Search Agents
- URL: https://arxiv.org/abs/2605.07039
- 重要度: 🔵Medium
- 要約:
  1. [課題] 進化的探索エージェントは固定プロンプト方策に依存し、高コスト評価下で探索戦略をタスク適応できない。
  2. [手法] 仮説生成・評価・選択を担う学習可能 advisor と実装を担う強力モデルを分離し、進化段階ごとに最適化を切り替える test-time RL を導入した。
  3. [結果] expert-parallel load balancing など複数課題で既存の frontier-model ベース探索法を上回り、収束速度と安定性を改善した。
- 一言メモ: マルチエージェントの探索ポリシーを実行モデルから分離する設計は、金融リサーチ agent の探索制御にも転用しやすい。

### [T2] Q-RAG: Long Context Multi-step Retrieval via Value-based Embedder Training
- URL: https://arxiv.org/abs/2511.07328
- 重要度: ⚡High
- 要約:
  1. [課題] 複雑質問では単発検索 RAG では不十分だが、既存の multi-step retrieval は小型 LLM の追加 FT が重い。
  2. [手法] LLM ではなく embedder 自体を RL で訓練し、長文脈 multi-step retrieval に最適化する Q-RAG を提案した。
  3. [結果] 10M tokens までの BabiLong と RULER で SOTA を達成し、計算効率の高い代替案を示した。
- 一言メモ: 社内文書・決算資料・XBRL を跨ぐ段階的検索の改善に直結し、ハルシネーション抑制と検索コスト削減の両方に効く。

### [T4] MatryoshkaLoRA: Learning Accurate Hierarchical Low-Rank Representations for LLM Fine-Tuning
- URL: https://arxiv.org/abs/2605.07850
- 重要度: ⚡High
- 要約:
  1. [課題] LoRA は rank を事前固定する必要があり、精度と効率のバランス探索に手間がかかる。
  2. [手法] LoRA adapter 間に対角行列を挿入して sub-rank 全体へ勾配情報を通し、階層的に rank 可変な MatryoshkaLoRA を学習する。
  3. [結果] 既存の rank-adaptive LoRA より全 rank 帯で精度効率トレードオフが良く、動的 rank 選択でも劣化が小さい。
- 一言メモ: GPU メモリ制約下で金融ドメイン SFT を回す際、rank 探索コストを減らしながら配備時の柔軟性を持てる。

### [T3] PrefixGuard: From LLM-Agent Traces to Online Failure-Warning Monitors
- URL: https://arxiv.org/abs/2605.06455
- 重要度: ⚡High
- 要約:
  1. [課題] ツール利用エージェントは失敗判定が最終時点まで遅れがちで、途中介入のための軽量監視が不足している。
  2. [手法] 生のトレースから typed-step を誘導する StepView と、prefix から失敗リスクを出す監視器を学習する PrefixGuard を提案した。
  3. [結果] WebArena などで raw-text や LLM judge を上回る警告性能を示し、早期警告の有効性を診断する枠組みも提示した。
- 一言メモ: 金融エージェントの誤発注・不正ツール呼び出し・逸脱行動を途中で止める監視層として実務価値が高い。

### [T2, T3] R^3-SQL: Ranking Reward and Resampling for Text-to-SQL
- URL: https://arxiv.org/abs/2604.25325
- 重要度: 🔵Medium
- 要約:
  1. [課題] Text-to-SQL は等価 SQL の不整合なランキングと、正解 SQL が候補に無い場合の復元不能性がボトルネックになる。
  2. [手法] 実行結果ベースで候補群をまとめて一貫性を評価し、正解欠如を検知すると agentic resampling する統合 reward を導入した。
  3. [結果] BIRD-dev で 75.03 execution accuracy を達成し、複数ベンチで安定した改善を示した。
- 一言メモ: 構造化出力スキーマ品質の改善という最重要課題に対し、候補再生成と実行整合性を組み合わせる発想が参考になる。

### [T1] Shallow Prefill, Deep Decoding: Efficient Long-Context Inference via Layer-Asymmetric KV Visibility
- URL: https://arxiv.org/abs/2605.06105
- 重要度: ⚡High
- 要約:
  1. [課題] 長文脈推論では prefill と全層 KV cache 保持が TTFT・TPOT・メモリを圧迫する。
  2. [手法] prefill token の KV を下位層にのみ保持し、decode token は全層可視のままにする layer-asymmetric KV policy を提案した。
  3. [結果] Llama-3.1-8B の 128K 文脈で品質をほぼ維持しつつ、TTFT 33%、TPOT 22%、active KV memory 25% を改善した。
- 一言メモ: PCIe 接続・量子化前提の長文脈 RAG で即効性があり、決算資料読解のレイテンシ削減候補として有望。

### [T3, T4] AEM: Adaptive Entropy Modulation for Multi-Turn Agentic Reinforcement Learning
- URL: https://arxiv.org/abs/2605.00425
- 重要度: ⚡High
- 要約:
  1. [課題] multi-turn agentic RL は outcome-only reward が疎で、長い軌跡への credit assignment が難しい。
  2. [手法] 中間報酬を足さず、応答レベルの不確実性 proxy で advantage を再重み付けして探索から活用へ移す AEM を提案した。
  3. [結果] ALFWorld、WebShop、SWE-bench-Verified で 1.5B から 32B の強力ベースラインを一貫して上回った。
- 一言メモ: 金融 agent の長手順タスクで dense reward を頑張って作らずに RL 改善できる点が実装上かなり現実的。

### [T1] SpecBlock: Block-Iterative Speculative Decoding with Dynamic Tree Drafting
- URL: https://arxiv.org/abs/2605.07243
- 重要度: ⚡High
- 要約:
  1. [課題] speculative decoding は path dependence を保つと drafter が遅く、並列化すると verifier に弾かれやすい。
  2. [手法] 1 回の drafter forward で複数位置を依存的に伸ばす block 単位の tree drafting と、分岐配分を決める rank head を導入した。
  3. [結果] EAGLE-3 比で drafting cost を 44-52% に抑えつつ平均速度向上を 8-13%、適応更新込みで 11-19% に伸ばした。
- 一言メモ: MoE ベース大規模モデルの推論速度改善に直結し、レイテンシ制約の厳しい対話型金融 agent と相性が良い。

### [T3] HyperEyes: Dual-Grained Efficiency-Aware Reinforcement Learning for Parallel Multimodal Search Agents
- URL: https://arxiv.org/abs/2605.07177
- 重要度: ⚡High
- 要約:
  1. [課題] 既存の検索 agent は対象ごとに逐次ツール呼び出しを行い、独立サブ検索でも無駄なラウンド数が増える。
  2. [手法] 複数エンティティを同時探索する並列検索 action を定義し、軌跡レベル reward と token レベル OPD を組み合わせた効率重視 RL を行う。
  3. [結果] 6 ベンチで強力な OSS agent を精度で 9.9% 上回り、tool-call rounds を平均 5.3 倍削減した。
- 一言メモ: 複数企業・指標・資料を並列に引く金融リサーチ workflow にそのまま移植したい設計思想。

### [T4] Flow-OPD: On-Policy Distillation for Flow Matching Models
- URL: https://arxiv.org/abs/2605.08063
- 重要度: 🔵Medium
- 要約:
  1. [課題] 多目的アライメント下の flow matching は scalar reward の疎さと目的間干渉で reward hacking が起きやすい。
  2. [手法] 単一報酬で鍛えた専門 teacher 群を作り、routing labeling と dense trajectory supervision で student に統合する OPD ベース後学習を提案した。
  3. [結果] Stable Diffusion 3.5 Medium 上で GenEval と OCR を大幅改善し、vanilla GRPO を約 10 点上回った。
- 一言メモ: 画像領域だが、専門 teacher を分けてから統合する設計は金融 agent の複数評価軸アライメントにも応用しやすい。

### [T1] Scaling Continual Learning to 300+ Tasks with Bi-Level Routing Mixture-of-Experts
- URL: https://arxiv.org/abs/2602.03473
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長いタスク列で安定性と可塑性を両立しつつ表現力を維持する continual learning は難しい。
  2. [手法] router 選択と expert routing の 2 段階で活性化を絞る bi-level routing MoE を提案した。
  3. [結果] 100 から 300 超のタスク列を含む設定で既存法を大きく上回り、超長系列 continual learning のスケーリングを示した。
- 一言メモ: 金融ドメインの継続学習や新タスク追加時の expert 分離設計として、MoE 運用の参考になる。

### [T3] SkCC: Portable and Secure Skill Compilation for Cross-Framework LLM Agents
- URL: https://arxiv.org/abs/2605.03353
- 重要度: 🔵Medium
- 要約:
  1. [課題] agent skill はフレームワークごとの prompt 依存が強く、単一 Markdown 配布では性能差と脆弱性が大きい。
  2. [手法] skill の意味表現を中間表現 SkIR に分離し、compile-time の Anti-Skill Injection 解析を加えるコンパイル基盤を提案した。

- 一言メモ: MCP/skill ベースの金融 agent を複数実行基盤で運用するなら、skill を資産化する基盤技術として重要。
