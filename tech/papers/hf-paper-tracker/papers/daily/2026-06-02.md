### [T3] Harness Updating Is Not Harness Benefit: Disentangling Evolution Capabilities in Self-Evolving LLM Agents
- URL: https://arxiv.org/abs/2605.30621
- 重要度: ⚡High
- 要約:
  1. [課題] プロンプト・スキル・メモリ・ツールから成る外部ハーネスを自己更新するエージェントで、更新能力と更新後に利益を得る能力の関係が不明だった。
  2. [手法] harness-updating と harness-benefit を分離し、複数能力帯のモデルで永続的ハーネス更新の効果を比較した。
  3. [結果] 更新生成能力はベースモデル能力に対してほぼ平坦で、更新を活用する能力は中位モデルで最大化する非単調な傾向を示した。
- 一言メモ: 金融エージェントでは「更新役」より「実行役」のハーネス呼び出し・長期指示追従を優先訓練すべきという設計指針になる。

  1. [課題] OPD で teacher-student disagreement が大きい token を優先しても、その差分が student に学習可能とは限らない。
  2. [手法] teacher が student の top-K 候補内に補正質量を置くかで token teachability を定義し、高 teachability 位置だけに OPD loss を適用した。
  3. [結果] Qwen2.5/Qwen3 の teacher-student 設定で、わずか5%の token を使う TA-OPD が full-token OPD や entropy/divergence baseline をしばしば上回った。
- 一言メモ: 大型MoEから金融特化小型モデルへ蒸留する際、全部のズレではなく学習可能なズレだけを拾うことで訓練効率を上げられる。
