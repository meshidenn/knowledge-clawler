### [T1/T4] Efficient Training on Multiple Consumer GPUs with RoundPipe
- URL: https://arxiv.org/abs/2604.27085
- 重要度: ⚡High
- 要約:
  1. [課題] PCIe 接続の複数 GPU で大規模 LLM を fine-tuning する際、既存 pipeline parallelism は重い層が特定 GPU に縛られて深いバブルが生じやすい。
  2. [手法] GPU を stateless worker のプールとして扱い、計算段を round-robin に動的再配置する RoundPipe と優先度付き転送・イベント同期・自動分割を組み合わせた。
  3. [結果] 8x RTX 4090 で既存法比 1.48-2.16 倍高速化し、Qwen3-235B の 31K 長文 LoRA まで単一サーバで回せることを示した。
- 一言メモ: NVLink なし PCIe 前提の現行環境にそのまま刺さる系統で、LoRA/QLoRA 学習の実行戦略を見直す価値が高い。

### [T2/T3] Operating-Layer Controls for Onchain Language-Model Agents Under Real Capital
- URL: https://arxiv.org/abs/2604.26091
- 重要度: ⚡High
- 要約:
  1. [課題] 実資金を扱う長期稼働エージェントでは、ベースモデル性能だけでは誤売買やポリシー逸脱を防げない。
  2. [手法] 自然言語戦略を typed controls、prompt compilation、policy validation、execution guard、trace-level observability を備えた operating layer で拘束した。
  3. [結果] 21 日の実運用で約 7.5M 回の agent invocation と約 30 万件の onchain action を処理し、特定 failure mode を大幅に削減しつつ資本配分率を改善した。
- 一言メモ: 金融 AI エージェントでは「良いモデル」より「型付き制御と監査可能な運用層」の方が信頼性を左右するという実証として極めて重要である。

### [T3] Claw-Eval-Live: A Live Agent Benchmark for Evolving Real-World Workflows
- URL: https://arxiv.org/abs/2604.28139
- 重要度: ⚡High
- 要約:
  1. [課題] 既存の agent benchmark は固定タスクと最終回答中心の採点に偏り、環境変化や実行証跡を十分に評価できない。
  2. [手法] 公開 workflow 需要信号からタスクを刷新し、trace・audit log・service state・workspace artifact に基づいて評価する live benchmark を設計した。

### [T4] Co-Evolving Policy Distillation
- URL: https://arxiv.org/abs/2604.27083
- 重要度: ⚡High
- 要約:
  1. [課題] 複数 expert 能力を単一モデルへ統合する際、混合 RLVR は能力干渉を起こし、後段 OPD は teacher-student の行動ギャップで吸収不全を起こしやすい。
  2. [手法] expert の RLVR を並行学習しながら、その途中から相互に bidirectional distillation させる CoPD を提案した。

- 一言メモ: 金融用サブ能力を個別学習してから 1 モデルへまとめる際、最後に寄せるのでなく学習中から共進化させる設計が参考になる。

- URL: https://arxiv.org/abs/2604.27251
- 重要度: 🔵Medium

  1. [課題] LLM に特定の推論形式を指示しても、実際にその reasoning schema を従わせられるかは明確でなかった。
