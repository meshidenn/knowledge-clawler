### [T1/T4] MASCing: Configurable Mixture-of-Experts Behavior via Activation Steering Masks
- URL: https://arxiv.org/abs/2604.27818
- 重要度: ⚡High
- 要約:
  1. [課題] MoEは推論効率が高い一方、専門家ルーティングに挙動が強く依存し、安全性や出力方針の制御が難しい。
  2. [手法] ルーティングlogitから挙動を予測する代理モデルを学習し、推論時に steering mask で専門家選択を上書きする軽量制御法を提案した。
  3. [結果] 7つの公開MoEで追加コストほぼ無しのまま、安全防御や特定挙動の誘導成功率を大きく改善した。
- 一言メモ: 金融向けMoE基盤で、再学習なしに安全制約や出力ポリシーを切り替える実装候補としてかなり実用的。

### [T2/T3] Web2BigTable: A Bi-Level Multi-Agent LLM System for Internet-Scale Information Search and Extraction
- URL: https://arxiv.org/abs/2604.27221
- 重要度: ⚡High
- 要約:
  1. [課題] Web探索エージェントは、単一対象の深掘りと多数対象の構造化集約を同時に高品質でこなすのが難しい。
  2. [手法] 上位オーケストレータと下位ワーカーからなる二層マルチエージェント構成に、run-verify-reflect ループと共有ワークスペースを組み合わせた。
  3. [結果] WideSearchで大幅なSOTA更新を達成し、深掘り系ベンチでも高い精度を示した。
- 一言メモ: 多銘柄・多ソースから表形式で根拠付き集約する設計そのもので、金融リサーチやM&A候補比較の基盤に直結する。

### [T3] From Context to Skills: Can Language Models Learn from Context Skillfully?
- URL: https://arxiv.org/abs/2604.27660
- 重要度: ⚡High
- 要約:
  1. [課題] 長く専門的な文脈から必要な規則や手順を学ぶには、手動スキル化のコストが高く、自動抽出にも外部フィードバックが乏しい。
  2. [手法] Challenger・Reasoner・Judge を中心とする自己進化型のマルチエージェント self-play で、文脈依存スキルを自律発見・精錬・選別する Ctx2Skill を提案した。
  3. [結果] CL-bench の4タスクで、抽出スキルを任意のLMへ差し込むだけで context learning 性能を一貫して改善した。
- 一言メモ: 社内文書や規制文書から実務スキルを自動抽出し、金融エージェントへ後付けする流れとして非常に相性が良い。

### [T4/T3] Hallucinations Undermine Trust; Metacognition is a Way Forward
- URL: https://arxiv.org/abs/2605.01428
- 重要度: ⚡High
- 要約:
  1. [課題] 幻覚は知識不足だけでなく、自分の不確実性を適切に認識・表明できないことが信頼性を損ねている。
  2. [手法] 幻覚を「自信を伴う誤り」と再定義し、答えるか黙るかではなく、不確実性を忠実に言語化する metacognition を中核原理として整理した。
  3. [結果] エージェントでは metacognition が「いつ検索し、何を信じるか」を司る制御層になるという研究課題を明確化した。
- 一言メモ: 構造化出力の品質改善でも、単なる正答率より「どこで不確実と宣言するか」の設計が重要だと示している。

### [T3/T4] HiL-Bench (Human-in-Loop Benchmark): Do Agents Know When to Ask for Help?
- URL: https://arxiv.org/abs/2604.09408
- 重要度: ⚡High
- 要約:
  1. [課題] 既存ベンチは仕様不足や矛盾がある実務状況を含まず、エージェントが「いつ質問すべきか」を測れていない。
  2. [手法] 曖昧さや欠落情報を段階的探索で発見させるタスク群と、質問精度と blocker 想起率を両立させる Ask-F1 指標を導入した。
  3. [結果] フロンティアモデルでも help-seeking 能力は大きく不足し、ただし Ask-F1 報酬を用いたRLで改善可能であることを示した。
- 一言メモ: 金融エージェントでも、曖昧な依頼を勝手に補完せず確認に戻る能力を明示的に訓練・評価すべき。

### [T2/T3] Hierarchical Abstract Tree for Cross-Document Retrieval-Augmented Generation
- URL: https://arxiv.org/abs/2605.00529
- 重要度: ⚡High
- 要約:
  1. [課題] 既存 Tree-RAG は単一文書前提が強く、複数文書に跨る multi-hop 質問で構造分離や粗い抽象化がボトルネックになる。
  2. [手法] 分布適応的な hierarchical abstract tree と、再構成クエリを使う multi-granular retrieval agent を組み合わせた Ψ-RAG を提案した。
  3. [結果] cross-document multi-hop QA で RAPTOR や HippoRAG 2 を大きく上回る平均F1を達成した。
- 一言メモ: 社内文書・XBRL・決算資料を跨ぐ金融RAGでは、単純chunk検索より階層索引と探索エージェントの併用が効きそう。

### [T3/T4] T^2PO: Uncertainty-Guided Exploration Control for Stable Multi-Turn Agentic Reinforcement Learning
- URL: https://arxiv.org/abs/2605.02178
- 重要度: ⚡High
- 要約:
  1. [課題] マルチターンRLでは、進捗に寄与しない低情報アクションが増えやすく、学習不安定化や崩壊を招く。
  2. [手法] 不確実性変化を監視して token 単位で thinking 介入し、turn 単位では探索進捗の薄い対話を再サンプルする T^2PO を提案した。
  3. [結果] WebShop、ALFWorld、Search QA で学習安定性と性能を改善し、探索効率の向上を示した。
- 一言メモ: 長手順の金融エージェント学習でも、「考える/進む/やり直す」を不確実性で制御する設計が有望。

### [T2] Prior-Aligned Data Cleaning for Tabular Foundation Models
- URL: https://arxiv.org/abs/2604.25154
- 重要度: 🔵Medium
- 要約:
  1. [課題] Tabular Foundation Model は実データの欠損・外れ値・重複で synthetic prior とズレ、精度と校正が同時に悪化する。
  2. [手法] クリーニング操作列をRLで最適化し、汚れた入力をモデルの事前分布へ近づける prior alignment として定式化した。
  3. [結果] 特定データセット群で TabPFN 精度を改善し、パラメトリックな cleaning action と事前学習方策の有効性を示した。
- 一言メモ: 財務テーブルや市場データ投入前の前処理を、静的ルールではなくモデル前提に合わせて最適化する発想として参考になる。

### [T3] From Skill Text to Skill Structure: The Scheduling-Structural-Logical Representation for Agent Skills
- URL: https://arxiv.org/abs/2604.24026
- 重要度: 🔵Medium
- 要約:
  1. [課題] SKILL.md のような自然言語中心のスキル定義は、検索・再利用・リスク審査に必要な実行構造が埋もれやすい。

- 一言メモ: MCP連携や社内ツール呼び出しを含む金融エージェント群では、スキル資産の監査可能性を高める土台になる。

### [T3] AcademiClaw: When Students Set Challenges for AI Agents
- URL: https://arxiv.org/abs/2605.02661
- 重要度: 🔵Medium
- 要約:
  1. [課題] 既存エージェントベンチは assistant 級タスク中心で、長期・複合・高難度の実務能力を十分に測れていない。
  2. [手法] 学生の実課題から収集した80件の長手順タスクを Docker sandbox 上で実行し、多面的 rubric と安全監査で採点した。

- 一言メモ: 金融向けでも、現場タスク由来の長手順評価セットを持たない限り本当の運用力は測れない。
