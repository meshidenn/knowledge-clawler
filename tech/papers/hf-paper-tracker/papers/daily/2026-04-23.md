### [T4] RDP LoRA: Geometry-Driven Identification for Parameter-Efficient Adaptation in Large Language Models
- URL: https://arxiv.org/abs/2604.19321
- 重要度: 🔵Medium
- 要約:
  1. [課題] LoRA の適用層選択が経験則頼みで、どの層を更新すべきかの構造的根拠が弱い。
  2. [手法] 隠れ状態の軌跡を幾何学的に捉え、RDP アルゴリズムで表現変化のブレークポイントとなる層だけを選んで LoRA を適用する。
  3. [結果] Qwen3-8B-Base で 13 層のみの適応でも MMLU-Math 81.67% を達成し、全36層 LoRA やランダム選択を上回った。
- 一言メモ: 金融 SFT/QLoRA で全層を触らず重要層だけに寄せれば、GPU メモリと学習時間を抑えつつ専門用語精度を上げる設計に直結する。

### [T3] AJ-Bench: Benchmarking Agent-as-a-Judge for Environment-Aware Evaluation
- URL: https://arxiv.org/abs/2604.18240
- 重要度: ⚡High
- 要約:
  1. [課題] 複雑な環境で動くエージェントを、静的ルールや LLM-as-a-Judge だけで検証すると汎化性が足りない。
  2. [手法] 検索・データシステム・GUI の3領域で、環境に実際に介入して証拠を集める Agent-as-a-Judge を評価する AJ-Bench を構築した。
  3. [結果] 従来の LLM-as-a-Judge より一貫して高性能だった一方、情報取得・状態検証・過程検証には未解決課題が多いことを示した。
- 一言メモ: 金融エージェントの注文前チェックやレポート検証を「ツールで証拠取得する審判エージェント」に分離する設計を強く後押しする。

### [T4] ShadowPEFT: Shadow Network for Parameter-Efficient Fine-Tuning
- URL: https://arxiv.org/abs/2604.19254
- 重要度: ⚡High
- 要約:
  1. [課題] LoRA 系は重みごとの局所摂動に依存するため、適応表現が分散しやすく転移や再利用の単位が弱い。
  2. [手法] 各層に並走する depth-shared な shadow module を導入し、重み空間ではなく層空間の refinement として PEFT を行う。
  3. [結果] 同等の学習パラメータ予算で LoRA/DoRA に匹敵または上回り、事前学習済み shadow module の再利用や detached mode も有効だった。
- 一言メモ: 金融ドメイン LoRA を量産するより、再利用可能な shadow module を軸にすると複数業務タスクへの横展開がしやすい。

### [T2/T4] Mitigating Multimodal Hallucination via Phase-wise Self-reward
- URL: https://arxiv.org/abs/2604.17982
- 重要度: 🔵Medium
- 要約:
  1. [課題] LVLM は視覚入力と不整合な幻覚を出しやすく、従来法は大量教師データか静的な後処理に依存していた。
  2. [手法] 幻覚が意味フェーズの切り替わり時に増える点を利用し、軽量 reward model で位相ごとに自己評価しながらデコードを補正する PSRD を提案した。
  3. [結果] LLaVA-1.5-7B の幻覚率を 50% 削減し、4モデル・5ベンチで既存の post-hoc 手法を上回った。
- 一言メモ: 財務表や IR 資料を扱う VLM 系でも、学習し直さず推論時ガードで幻覚を抑える発想はそのまま転用できる。

### [T1] Speculative Decoding for Autoregressive Video Generation
- URL: https://arxiv.org/abs/2604.17397
- 重要度: ⚪Low
- 要約:
  1. [課題] 推論高速化の本命である speculative decoding は、連続値テンソルを出す自己回帰 video diffusion にはそのまま適用しづらい。
  2. [手法] 小型 drafter が提案したブロックを ImageReward ベースの品質ルータで検証し、良い候補だけ大モデルの KV cache に受け渡す SDVG を設計した。
  3. [結果] 14B ターゲットに対して品質を約98%維持しつつ 1.59 倍、設定次第で 2.09 倍まで高速化した。
- 一言メモ: ドメインは動画だが、「品質ルータ付き speculative decoding」という構成は MoE 推論や長文生成の低遅延化検討に示唆がある。

### [T4] TEMPO: Scaling Test-time Training for Large Reasoning Models
- URL: https://arxiv.org/abs/2604.19295
- 重要度: 🔵Medium
- 要約:
  1. [課題] Test-time training は追加計算を増やしても報酬信号のドリフトで頭打ちし、多様性崩壊も起こしやすい。
  2. [手法] ラベルなしテスト問題で方策更新しつつ、ラベル付きデータで critic を定期再較正する EM 風の交互最適化 TEMPO を提案した。
  3. [結果] OLMO3-7B の AIME 2024 を 33.0% から 51.1%、Qwen3-14B を 42.3% から 65.8% へ改善し、多様性も維持した。
- 一言メモ: 金融QAやバリュエーション推論でも、本番近傍データに対する小規模な継続適応を安全に回す設計として参考になる。

### [T3] Chat2Workflow: A Benchmark for Generating Executable Visual Workflows with Natural Language
- URL: https://arxiv.org/abs/2604.19667
- 重要度: 🔵Medium
- 要約:
  1. [課題] 業務ワークフローは信頼性が高い一方で、自然言語からそのまま実行可能な形へ落とすのは依然として難しい。
  2. [手法] Dify や Coze にデプロイ可能な実業務フローを基に、自然言語から実行可能 workflow を生成する Chat2Workflow ベンチと補助エージェントを作成した。
  3. [結果] 先端モデルでも複雑要件や変更要件下では安定した workflow 生成が難しく、補助エージェントで改善しても解決率の伸びは限定的だった。
- 一言メモ: 金融オペレーションを GUI/フローで管理する場合、自然言語だけに頼らず中間 DSL や検証器が必要だと再確認できる。

### [T2] Code-Switching Information Retrieval: Benchmarks, Analysis, and the Limits of Current Retrievers
- URL: https://arxiv.org/abs/2604.17632
- 重要度: 🔵Medium
- 要約:
  1. [課題] 多言語 IR は進んでも、実務で頻出する code-switching クエリへの耐性は十分に評価されていない。
  2. [手法] 人手注釈による CSR-L と 11 タスクから成る CS-MTEB を整備し、統計・dense・late interaction 系の検索器を横断比較した。
  3. [結果] code-switching により最大 27% の性能低下が起き、埋め込み空間の乖離が主因で、単純な語彙拡張では解消しきれなかった。
- 一言メモ: 英日混在の決算・社内文書検索では現行 multilingual embedding を過信せず、混在クエリ専用評価セットを先に作るべきだ。

### [T2] Dual-View Training for Instruction-Following Information Retrieval
- URL: https://arxiv.org/abs/2604.18845
- 重要度: ⚡High
- 要約:
  1. [課題] 多くの retriever は意味的関連性には強いが、「除外条件」「属性条件」など命令に従う検索が苦手である。
  2. [手法] 同じ文書ペアに対して relevance が反転する補完命令を LLM で合成し、命令そのものを見ないと判別できない学習信号を与える dual-view training を提案した。
  3. [結果] 305M encoder で FollowIR を 45% 改善し、同規模以上の汎用 embedding モデルを上回った。
- 一言メモ: 「特定会計基準のみ」「最新四半期を優先」など条件付き金融 RAG にそのまま効く、今日いちばん実装に近い検索改善案。

### [T3] Symbolic Guardrails for Domain-Specific Agents: Stronger Safety and Security Guarantees Without Sacrificing Utility
- URL: https://arxiv.org/abs/2604.15579
- 重要度: ⚡High
- 要約:
  1. [課題] 高リスク業務で使うエージェントは、学習ベースの安全化だけでは強い保証を出せない。
  2. [手法] 80件の安全ベンチを分析したうえで、どの方針が symbolic guardrails で強制可能かを整理し、複数ベンチで効用との両立を検証した。
  3. [結果] 具体的方針の 74% は比較的単純な symbolic guardrails で強制可能で、安全性を上げつつ成功率を落としにくかった。
- 一言メモ: 金融領域では「許可された銘柄・顧客・操作だけ実行」のようなルールを LLM 任せにせず、記号的制約で囲うのが現実的。

### [T1] River-LLM: Large Language Model Seamless Exit Based on KV Share
- URL: https://arxiv.org/abs/2604.18396
- 重要度: ⚡High
- 要約:
  1. [課題] Early Exit は理論上速いが、decoder-only LLM ではスキップ層の KV cache 欠損が壁になり実効速度が出にくい。
  2. [手法] 軽量な KV-Shared Exit River で欠損 KV を自然生成し、状態遷移類似度から累積誤差を見積もって exit を制御する training-free 手法を提案した。
  3. [結果] 数学推論とコード生成で品質を保ちながら 1.71〜2.16 倍の実効高速化を達成した。
- 一言メモ: PCIe 前提の大規模 MoE 推論でボトルネックになりやすいレイテンシ改善に直結し、量子化と併用候補として有望。

### [T2] On the Robustness of LLM-Based Dense Retrievers: A Systematic Analysis of Generalizability and Stability
- URL: https://arxiv.org/abs/2604.16576
- 重要度: 🔵Medium
- 要約:
  1. [課題] LLM-based dense retriever は高性能化しているが、データセット跨ぎの汎化性と入力ゆらぎへの頑健性が十分検証されていない。
  2. [手法] 30 データセットにまたがる評価と、typo・言い換え・汚染攻撃に対する安定性分析を組み合わせて、一般化性能と安定性を分解した。
  3. [結果] instruction-tuned 系は強い一方、複雑推論向け最適化モデルには汎化税があり、typo には強くても意味的摂動には依然弱かった。
- 一言メモ: 社内 RAG の retriever 選定ではベンチスコアだけでなく、言い換えや文体差への安定性試験を必須にすべきだ。
