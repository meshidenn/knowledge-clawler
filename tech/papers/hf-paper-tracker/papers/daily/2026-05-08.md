### [T2/T3] OpenSearch-VL: An Open Recipe for Frontier Multimodal Search Agents
- URL: https://arxiv.org/abs/2605.05185
- 重要度: ⚡High
- 要約:
  1. [課題] フロンティア級の深い検索エージェントは有力だが、再現可能な学習データ・ツール環境・RLレシピが公開されていない。
  2. [手法] Wikipedia経路サンプリングと視覚アンカー付きデータ合成、OCRや画像補正を含む多様なツール環境、失敗後トークンを扱う fatal-aware GRPO を統合した。
  3. [結果] 7つのベンチで平均10点超の改善を示し、一部では商用級モデルに迫るオープンなマルチモーダル検索エージェントを実現した。
- 一言メモ: 決算資料・IRスライド・表画像まで含む金融リサーチでは、検索器より上の「探索軌跡とツール設計」まで公開している点がかなり実用的。

### [T2/T3] Rethinking Reasoning-Intensive Retrieval: Evaluating and Advancing Retrievers in Agentic Search Systems
- URL: https://arxiv.org/abs/2605.04018
- 重要度: ⚡High
- 要約:
  1. [課題] 既存retriever評価は単一文書の関連性に寄りすぎており、エージェント検索で必要な補完的証拠集合を測れていない。
  2. [手法] multi-aspect gold evidence を持つ BRIGHT-Pro と、補完的positive・条件付きhard negative を作る RTriever-Synth で評価と学習を再設計した。
  3. [結果] aspect-aware な評価で既存指標では見えない差を可視化し、Qwen3-Embedding-4Bベースの RTriever-4B を有意に改善した。
- 一言メモ: 社内文書・XBRL・決算短信を束ねる金融RAGでは、「1件当てる」より「必要な証拠束を揃える」retriever設計へ寄せるべきだと分かる。

### [T2/T3] Chain of Evidence: Pixel-Level Visual Attribution for Iterative Retrieval-Augmented Generation
- URL: https://arxiv.org/abs/2605.01284
- 重要度: ⚡High
- 要約:
  1. [課題] text化前提のiRAGは、図表やレイアウトを含むPDFで根拠位置の追跡と視覚的意味を失いやすい。
  2. [手法] 取得文書のスクリーンショットをVLMで直接処理し、推論連鎖に対応する bbox を返す retriever-agnostic な visual attribution を提案した。
  3. [結果] Wiki-CoE と SlideVQA で、視覚レイアウト理解が必要な設定において text-based baseline を上回った。
- 一言メモ: IR資料や決算説明会スライドで「どの図表を根拠にしたか」まで出せるので、金融RAGの監査性を大きく上げられる。

### [T3] ARIS: Autonomous Research via Adversarial Multi-Agent Collaboration
- URL: https://arxiv.org/abs/2605.03042
- 重要度: ⚡High
- 要約:
  1. [課題] 長時間動く研究エージェントは、もっともらしいが根拠不十分な主張を静かに積み上げやすい。
  2. [手法] 実行役と異系統レビュアを常設し、skills・MCP・永続wiki・claim auditing を3層の harness として統合した。

- 一言メモ: 金融AIエージェントでも、実行モデルとは別に監査モデルと根拠台帳を持たせる構成が必須に近い。

### [T3/T4] Reinforcement Learning for LLM-based Multi-Agent Systems through Orchestration Traces
- URL: https://arxiv.org/abs/2605.02801
- 重要度: ⚡High
- 要約:
  1. [課題] マルチエージェントの学習は、個々の行動だけでなく spawn・delegate・aggregate・stop の最適化単位が整理されていない。
  2. [手法] orchestration trace を時間グラフとして捉え、報酬設計・credit単位・5つの下位意思決定へ分解して文献地図とJSON schemaを整備した。

- 一言メモ: 自社オーケストレータでも、停止判断と委譲品質を独立した学習・評価対象に切り出す価値が高い。

### [T3] OpenSeeker-v2: Pushing the Limits of Search Agents with Informative and High-Difficulty Trajectories
- URL: https://arxiv.org/abs/2605.04036
- 重要度: ⚡High
- 要約:
  1. [課題] 高性能search agentは通常、CPT・SFT・RLを含む重い産業パイプラインに依存し、学術側で再現しにくい。
  2. [手法] 知識グラフ拡張、ツール数拡大、低品質短手順の厳格除外で、高情報量・高難度の探索軌跡を合成した。
  3. [結果] 10.6k件のSFTだけで BrowseComp など4ベンチで同規模帯SOTAを達成し、重いRLパイプラインに対抗した。
- 一言メモ: 金融リサーチエージェントも、まずは高難度で情報密度の高い探索軌跡を集める方がRLより先に効きそうだ。

### [T3/T4] HiL-Bench (Human-in-Loop Benchmark): Do Agents Know When to Ask for Help?
- URL: https://arxiv.org/abs/2604.09408
- 重要度: ⚡High
- 要約:
  1. [課題] frontier agent は不完全仕様下で「自律で進めるべきか、人に聞くべきか」の判断が弱いのに、既存ベンチはそこを測っていない。
  2. [手法] human-validated blocker を含むタスクと、質問の精度と blocker recall を両立させる Ask-F1 指標を導入した。
  3. [結果] 主要モデルは選択的エスカレーション能力が低く、ただし shaped reward によるRLで help-seeking は改善可能だと示した。
- 一言メモ: 金融実務では曖昧な条件を勝手に補完するコストが高いため、「確認に戻る能力」を明示的に鍛える必要がある。

### [T4] The First Token Knows: Single-Decode Confidence for Hallucination Detection
- URL: https://arxiv.org/abs/2605.05166
- 重要度: ⚡High
- 要約:
  1. [課題] self-consistency による幻覚検知は複数回デコードが必要で、速度とコストの両面で重い。
  2. [手法] 最初の内容語トークンの top-K logit エントロピーから単発推論の信頼度指標 phi_first を計算した。
  3. [結果] 7B〜8B級モデルの factual QA で semantic self-consistency と同等以上の AUROC を、1回デコードで達成した。
- 一言メモ: 構造化出力前に低コストな不確実性シグナルを差し込めるので、再質問・再検索・人手確認のトリガとして使いやすい。

### [T4/T2] How Fast Should a Model Commit to Supervision? Training Reasoning Models on the Tsallis Loss Continuum
- URL: https://arxiv.org/abs/2604.25907
- 重要度: ⚡High
- 要約:
  1. [課題] 出力レベル報酬しかない新規タスク適応では、初期成功率が低いと RLVR が cold start から抜け出せない。
  2. [手法] RLVR と latent trajectory の尤度最大化を連続的につなぐ Tsallis 損失族を導入し、GARL と PAFT の2推定器で近似した。
  3. [結果] FinQA・HotPotQA・MuSiQueで cold-start 停滞を大きく緩和し、GRPO が失敗する条件でも学習を進めた。
- 一言メモ: 金融QAや構造化推論の後学習では、SFTとRLの中間にあるこうした橋渡し損失がかなり実践的。

### [T4] ResRL: Boosting LLM Reasoning via Negative Sample Projection Residual Reinforcement Learning
- URL: https://arxiv.org/abs/2605.00380
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLVRは正例報酬へ寄りすぎて生成多様性が落ち、負例強化も正負で共有される有用表現まで潰しがちである。
  2. [手法] 負例表現を正例低ランク部分空間へ射影し、その残差だけで負勾配を調整する ResRL を提案した。
  3. [結果] 数学・コード・agent task・function calling を含む12ベンチ平均で強い baseline を上回り、多様性を保ったまま性能を伸ばした。
- 一言メモ: 金融向けDPO/RLでも、誤答罰則を強めすぎて出力の幅を失う問題に対する実装ヒントになる。

### [T3] WindowsWorld: A Process-Centric Benchmark of Autonomous GUI Agents in Professional Cross-Application Environments
- URL: https://arxiv.org/abs/2604.27776
- 重要度: 🔵Medium
- 要約:
  1. [課題] GUI agent 評価は単一アプリの孤立タスクに偏り、実務のような複数アプリ横断ワークフローを十分測れていない。
  2. [手法] 16職種を起点に人手洗練した cross-application タスク181件を、17種のデスクトップアプリ上で構築した。
