### [T1] Temporally Extended Mixture-of-Experts Models
- URL: https://arxiv.org/abs/2604.20156
- 重要度: ⚡High
- 要約:
  1. [課題] 通常の MoE は token ごとに expert を切り替えるため、GPU メモリを超えると offload や prefetch の効率が急激に悪化する。
  2. [手法] option-critic と deliberation cost を用いた controller を各層に追加し、expert 集合をいつ切り替えるかを学習する temporally extended MoE を提案した。
  3. [結果] gpt-oss-20b で expert switch rate を 50% 超から 5% 未満へ抑えつつ、MATH・MMLU・MMMLU で base 性能の最大 90% を維持した。
- 一言メモ: PCIe 接続前提の Expert Parallelism では通信とロードの揺れを直接減らせるため、現行の推論制約に最も近い。

### [T1] Expert Upcycling: Shifting the Compute-Efficient Frontier of Mixture-of-Experts
- URL: https://arxiv.org/abs/2604.19835
- 重要度: ⚡High
- 要約:
  1. [課題] 大規模 MoE を最初から学習すると総パラメータ増加に伴うメモリ負荷と通信コストが重く、学習費用が高すぎる。
  2. [手法] 学習済み E-expert MoE を expert duplication と router extension で mE-expert に拡張し、継続事前学習で専門化させる expert upcycling を提案した。
  3. [結果] 7B-13B 規模で fixed-size baseline と同等の validation loss を保ちながら GPU 時間を 32% 削減し、utility-based duplication で短い CPT でも差を詰めた。
- 一言メモ: 金融向け追加学習の前に既存 MoE を段階拡張する筋が見え、ゼロから巨大モデルを回すより現実的である。

### [T2] Contexts are Never Long Enough: Structured Reasoning for Scalable Question Answering over Long Document Sets
- URL: https://arxiv.org/abs/2604.22294
- 重要度: ⚡High
- 要約:
  1. [課題] 長大な文書集合 QA は chunking 後の evidence 集約がボトルネックになり、context window を増やしても本質的には解決しない。
  2. [手法] 重要情報を関係データベースへ抽出し、provenance と rationale を使う data reconciliation を挟んで SQL 上で推論する SLIDERS を提案した。
  3. [結果] 既存の long-context benchmark で平均 6.6 点 GPT-4.1 を上回り、3.9M/36M token の新 benchmark でも次善法を大きく上回った。
- 一言メモ: XBRL・決算資料・社内文書を跨ぐ RAG は「長文を読む」より「構造化 state に落とす」方が筋が良いと示している。

### [T2/T4] LLaTiSA: Towards Difficulty-Stratified Time Series Reasoning from Visual Perception to Semantics
- URL: https://arxiv.org/abs/2604.17295
- 重要度: ⚡High
- 要約:
  1. [課題] LLM の時系列理解はタスク定義と評価軸が断片的で、複雑な temporal reasoning を一貫して鍛えにくい。
  2. [手法] 4 段階 taxonomy と 8.3 万件の HiTSR を整備し、チャート表現と数表を併用する multi-stage curriculum fine-tuning で TSRM を構築した。
  3. [結果] 多様な時系列推論ベンチで強い性能を示し、実世界シナリオを含む OOD 一般化でも優位だった。
- 一言メモ: 価格推移や業績推移を図表混在で読む金融エージェントの学習設計に直結する。

### [T2] Learning Evidence Highlighting for Frozen LLMs
- URL: https://arxiv.org/abs/2604.22565
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長くノイジーな文脈では、LLM が決定的証拠を見落として推論を外すことが多い。
  2. [手法] 入力を書き換えず最小限の highlight tag を付ける Emphasis Actor を RL で学習し、frozen Solver とは分離して最適化する HiLight を提案した。
  3. [結果] 長文 QA と sequential recommendation で prompt 系 baseline を上回り、未見 Solver 群にも zero-shot 転移した。
- 一言メモ: 決算資料や規制文書の重要 span を強調してから推論させる前処理として試しやすい。

### [T2] Trust but Verify: Introducing DAVinCI -- A Framework for Dual Attribution and Verification in Claim Inference for Language Models
- URL: https://arxiv.org/abs/2604.21193
- 重要度: 🔵Medium
- 要約:
  1. [課題] 高リスク領域では生成文の検証だけでなく、各 claim がどの内部根拠・外部根拠に基づくか追跡できないと監査性が足りない。
  2. [手法] claim を内部コンポーネントと外部ソースへ帰属させた後、entailment と confidence calibration で検証する二段階の DAVinCI を提案した。
  3. [結果] FEVER 系データで accuracy と attribution precision/recall/F1 を 5〜20% 改善した。
- 一言メモ: 投資判断文や要約レポートを claim 単位で検証したいなら、そのまま設計テンプレになる。

### [T2/T4] COMPASS: COntinual Multilingual PEFT with Adaptive Semantic Sampling
- URL: https://arxiv.org/abs/2604.20720
- 重要度: 🔵Medium
- 要約:
  1. [課題] 多言語適応では naive fine-tuning により負の言語間干渉が起きやすく、継続運用中の分布変化にも弱い。

### [T3] Memanto: Typed Semantic Memory with Information-Theoretic Retrieval for Long-Horizon Agents
- URL: https://arxiv.org/abs/2604.22085
- 重要度: ⚡High
- 要約:
  1. [課題] 長期エージェントの memory は graph 構築や多段 retrieval の運用コストが高く、低遅延化が難しい。
  2. [手法] 13 種の typed semantic memory、矛盾解決、temporal versioning を備えた universal memory layer を、no-indexing の情報理論検索で支えた。
  3. [結果] LongMemEval 89.8%、LoCoMo 87.1% で既存 graph/vector 系を上回り、sub-90ms の単発 retrieval と ingestion delay なしを実現した。
- 一言メモ: 顧客履歴や案件文脈を持つ金融エージェントでは、複雑な KG より低遅延 typed memory の方が先に効きそうである。

### [T3] VLAA-GUI: Knowing When to Stop, Recover, and Search, A Modular Framework for GUI Automation
- URL: https://arxiv.org/abs/2604.21375
- 重要度: ⚡High
- 要約:
  1. [課題] GUI エージェントは早すぎる完了宣言と失敗ループが多く、実務自動化で信頼性が不足する。

### [T4] Abstain-R1: Calibrated Abstention and Post-Refusal Clarification via Verifiable RL
- URL: https://arxiv.org/abs/2604.17073
- 重要度: ⚡High
- 要約:
  1. [課題] reasoning 強化後の LLM は不明な問いにも推測で答えがちで、単なる拒否ではなく「何が足りないか」を示せない。
  2. [手法] answerable 問題では正答を、unanswerable 問題では abstention と missing information の明示を同時に報酬化する clarification-aware RLVR を設計した。
  3. [結果] 3B の Abstain-R1 は Abstain-Test、Abstain-QA、SelfAware で abstention と clarification を大きく改善しつつ、回答可能問題の性能を維持した。
- 一言メモ: 金融でのハルシネーション抑制は「答えない」だけでなく「追加で何を取得すべきか」を返す設計が重要である。

### [T4] Hybrid Policy Distillation for LLMs
- URL: https://arxiv.org/abs/2604.20244
- 重要度: 🔵Medium
- 要約:
  1. [課題] LLM 蒸留は divergence の向きや data regime によって安定性と性能が大きくぶれやすい。
  2. [手法] forward KL と reverse KL を併用し、off-policy data と軽量な近似 on-policy sampling を組み合わせる Hybrid Policy Distillation を提案した。
  3. [結果] 数学推論、対話、コード生成で最適化安定性、計算効率、最終性能の全てを改善した。
- 一言メモ: 大規模教師から金融業務用の軽量補助モデルを作る蒸留工程の改善候補として現実的である。
