### [T1] GQLA: Group-Query Latent Attention for Hardware-Adaptive Large Language Model Decoding
- URL: https://arxiv.org/abs/2605.15250
- 重要度: ⚡High
- 要約:
  1. [課題] DeepSeek系 MLA は高帯域 GPU では強いが、H20 級や head 軸並列を使いたい環境では推論経路が固定で扱いにくい。
  2. [手法] 同一重みから MQA 吸収経路と GQA 経路を切り替えられる GQLA を提案し、ハードウェアに応じて最適な decode path を選べるようにした。
  3. [結果] LLaMA-3-8B 変換実験で GQA の柔軟性を保ちつつ、MQA 経路では KV cache を大幅圧縮し、ゼロ冗長 tensor parallel にも対応した。
- 一言メモ: PCIe 環境での推論最適化に直結しており、単一重みで複数 GPU 条件へ適応できる発想はかなり実務的。

### [T1] HodgeCover: Higher-Order Topological Coverage Drives Compression of Sparse Mixture-of-Experts
- URL: https://arxiv.org/abs/2605.13997
- 重要度: 🔵Medium
- 要約:
  1. [課題] 既存の learning-free MoE 圧縮は expert 間の pairwise 類似だけを見るため、3者以上で起きる非可換な競合を見落としやすい。
  2. [手法] expert 関係を 2-complex として表現し、Hodge 分解で triple 単位の圧縮阻害構造を抽出して選択に反映した。
  3. [結果] 強い expert 削減条件で既存法と同等以上を達成し、特に aggressive compression 領域で有利だった。
- 一言メモ: RTX 6000 Pro 前提で expert 数を削って実運用へ寄せるとき、単純な pruning より構造保存型圧縮が効く可能性がある。

### [T1] Long Context Pre-Training with Lighthouse Attention
- URL: https://arxiv.org/abs/2605.06554
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長文事前学習は full attention の二乗コストが重く、長い決算資料や社内文書を食わせる継続学習が高価になりやすい。
  2. [手法] 学習中だけ系列を階層圧縮する Lighthouse Attention を外付けし、終盤で full attention に戻す二段階学習を採用した。
  3. [結果] 小規模事前学習で総学習時間を短縮しつつ、最終 loss も full attention 学習より改善した。
- 一言メモ: 長文 RAG だけでなく長文事前学習そのもののコスト圧縮手段として検討価値がある。

### [T2] MLAIRE: Multilingual Language-Aware Information Retrieval Evaluation Protocal
- URL: https://arxiv.org/abs/2605.07249
- 重要度: ⚡High
- 要約:
  1. [課題] 多言語検索では「意味的に関連している」だけでなく、ユーザーが読める言語で返すこと自体が実用性を左右するが、既存評価はそこを測れていない。
  2. [手法] 並列文書プール上で semantic retrieval と query-language preference を分離評価する MLAIRE と、LPR や Lang-nDCG などの指標を導入した。
  3. [結果] 31 種の dense・sparse・late-interaction retriever を比べると、通常指標では見えない「意味は合うが言語がズレる」失敗が明確に表れた。
- 一言メモ: 多言語決算書や海外 IR 資料を扱う RAG では、単なる recall ではなく言語整合性の評価軸を追加すべきだと分かる。

### [T2] CiteVQA: Benchmarking Evidence Attribution for Trustworthy Document Intelligence
- URL: https://arxiv.org/abs/2605.12882
- 重要度: ⚡High
- 要約:
  1. [課題] Doc-VQA は最終回答だけを採点しがちで、正答でも根拠箇所を誤る attribution hallucination を見逃してしまう。
  2. [手法] 回答に加えて根拠 bounding box の引用を必須にする CiteVQA と、回答と引用が両方正しい場合だけ加点する Strict Attributed Accuracy を提案した。
  3. [結果] 20 モデル監査で性能上位モデルでも根拠一致は大きく崩れ、最良システムでも SAA は 76.0 に留まった。
- 一言メモ: 金融文書 QA で最重要の「どのページ・どの表セルを根拠にしたか」を評価に入れるべきだという強い根拠になる。

### [T2] WildTableBench: Benchmarking Multimodal Foundation Models on Table Understanding In the Wild
- URL: https://arxiv.org/abs/2605.01018
- 重要度: ⚡High
- 要約:
  1. [課題] 実世界の表画像はレイアウト崩れや複雑な数値関係を含むが、既存評価は整った表データに偏っていた。
  2. [手法] 多様な実画像表 402 枚と 928 問からなる in-the-wild 表理解ベンチマークを構築した。
  3. [結果] 21 モデル中 50% を超えたのは 1 モデルだけで、構造把握と数値推論の弱さが大きく残った。
- 一言メモ: IR スライドや決算説明資料の表理解そのもので、金融文書 VLM 評価セット設計の参考になる。

### [T2] PreScam: A Benchmark for Predicting Scam Progression from Early Conversations
- URL: https://arxiv.org/abs/2605.12243
- 重要度: 🔵Medium
- 要約:
  1. [課題] 投資詐欺のような対話型リスクは段階的に進行するが、既存研究は静的分類に寄りすぎて進行予測を扱えていなかった。
  2. [手法] 11,573 件の実詐欺会話を scam lifecycle に沿って整理し、終盤接近予測と次アクション予測の 2 タスクを定義した。
  3. [結果] 終了予測では教師あり encoder がゼロショット LLM を上回り、次アクション予測は強い LLM でも中程度に留まった。
- 一言メモ: 金融 AI の不正検知や営業対話モニタでも、単発ラベル分類より時系列リスク進行モデルが重要だと示す。

### [T3] Auditing Agent Harness Safety
- URL: https://arxiv.org/abs/2605.14271
- 重要度: ⚡High
- 要約:
  1. [課題] エージェント安全性評価が最終回答中心だと、途中の権限逸脱や情報漏えいのような harness 起因の事故を捉えられない。
  2. [手法] 軌跡全体を境界遵守・実行忠実性・安定性で監査する HarnessAudit と、210 タスクの HarnessAudit-Bench を構築した。
  3. [結果] task completion と安全実行は大きくズレており、特に resource access と inter-agent information transfer に違反が集中した。
- 一言メモ: 金融エージェントで最終回答が正しくても権限境界を越えたら失格なので、まさに評価設計へ直結する。

### [T3/T4] MetaAgent-X : Breaking the Ceiling of Automatic Multi-Agent Systems via End-to-End Reinforcement Learning
- URL: https://arxiv.org/abs/2605.14212
- 重要度: ⚡High
- 要約:
  1. [課題] 既存の automatic MAS は designer だけ最適化して executor を凍結するなど、自己設計と自己実行の end-to-end 学習が不足していた。
  2. [手法] script ベースの MAS 生成、実行 rollout、designer/executor 双方への credit assignment を統合した end-to-end RL 枠組みを提案した。
  3. [結果] 既存自動 MAS ベースラインを最大 21.7% 上回り、designer と executor が段階的に共進化することを示した。
- 一言メモ: 金融向け multi-agent orchestration でも、固定ワークフローではなく編成自体を学習させる方向が見えてくる。

### [T4] Hölder Policy Optimisation
- URL: https://arxiv.org/abs/2605.12058
- 重要度: ⚡High
- 要約:
  1. [課題] GRPO 系学習では token 確率の集約方法が固定されるため、勾配集中と分散安定性のトレードオフをうまく制御できない。
  2. [手法] 集約を Hölder mean で一般化し、学習フェーズに応じて p を動的にアニールする HölderPO を提案した。
  3. [結果] 数学ベンチ群で平均 54.9% の SOTA を達成し、標準 GRPO 比で 7.2% 相対改善、ALFWorld でも 93.8% を記録した。
- 一言メモ: 構造化出力の RL 改善でも、報酬設計だけでなく token 集約の安定化が効く可能性が高い。

### [T4] Distilling Long-CoT Reasoning through Collaborative Step-wise Multi-Teacher Decoding
- URL: https://arxiv.org/abs/2605.02290
- 重要度: ⚡High
- 要約:
  1. [課題] Long-CoT 蒸留は完成済み推論列の事後選別に偏り、異種 teacher の補完性や途中分岐の探索を活かせていなかった。
  2. [手法] 複数 teacher が step-wise に協調して推論列を構築する CoRD を提案し、予測 perplexity と beam search で候補を絞った。
  3. [結果] より高品質な reasoning データを生成し、少ない supervision で student が teacher 近傍の性能に到達した。
- 一言メモ: 金融 reasoning の蒸留でも、単一教師より複数強モデルの逐次合成でデータ品質を上げられそうだ。

### [T4] Learning to Foresee: Unveiling the Unlocking Efficiency of On-Policy Distillation
- URL: https://arxiv.org/abs/2605.11739
- 重要度: 🔵Medium
- 要約:
  1. [課題] On-policy distillation がなぜ効率的かは知られていたが、どのモジュールへどう更新が集まるかの力学が不明だった。
  2. [手法] OPD が早期に最終更新方向を先取りするという foresight 仮説を立て、そこから extrapolation で加速する EffOPD を導出した。
  3. [結果] 追加モジュールなしで OPD を平均 3 倍高速化しつつ、最終性能をほぼ維持した。
- 一言メモ: 高価な post-training を回す前に、更新方向の先読みで反復回数を削る発想は実運用コストに効く。

### [T4] Forgetting That Sticks: Quantization-Permanent Unlearning via Circuit Attribution
- URL: https://arxiv.org/abs/2605.15138
- 重要度: ⚡High
- 要約:
  1. [課題] 4-bit 量子化後に unlearning が崩れることがあり、full precision 時点の忘却評価だけでは実運用安全性を保証できない。
  2. [手法] 回路寄与解析で forget 対象の最小部分回路を絞り、量子化 bin 幅を超える更新を保証する MANSU を提案した。
  3. [結果] 忘却・保持・PTQ 後の非悪化・構造的消去の 4 条件を同時に満たし、既存勾配法の PTQ 逆戻りを克服した。
- 一言メモ: AWQ/INT4 前提の本プロジェクトでは、量子化後に効く安全化や知識削除だけを信用すべきだと示している。
