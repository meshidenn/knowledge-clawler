### [T4] Reinforcement Learning via Value Gradient Flow
- URL: https://arxiv.org/abs/2604.14265
- 重要度: 🔵Medium
- 要約:
  1. [課題] 挙動正則化つきRLでは、既存の方策勾配法は大規模生成モデルに載せにくく、拒否サンプリングは保守的すぎる。
  2. [手法] 参照分布から価値誘導された最適分布への輸送を最適輸送問題として定式化し、離散的な gradient flow で更新する VGF を提案した。
  3. [結果] offline RL と LLM RL の両方で既存法を上回り、輸送予算で正則化強度とテスト時スケーリングを制御できた。
- 一言メモ: DPO/KTO の先で RL を使う場合に、明示的な方策パラメタ化を弱めつつ安定化できる可能性がある。

### [T3] RadAgent: A tool-using AI agent for stepwise interpretation of chest computed tomography
- URL: https://arxiv.org/abs/2604.15231
- 重要度: 🔵Medium
- 要約:
  1. [課題] VLM 単体の最終レポート生成は、中間判断が見えず、検証可能性と信頼性が不足しやすい。
  2. [手法] CT 読影をツール利用つきの段階的エージェント処理に分解し、各判断とツール呼び出しのトレースを明示した。
  3. [結果] 3D VLM ベースラインより精度・頑健性・faithfulness が改善し、可観測な推論過程を実現した。
- 一言メモ: 金融レポート生成でも、最終回答だけでなく「取得データ→計算→根拠」の逐次トレース設計が重要だと再確認できる。

### [T2 / T3] Don't Retrieve, Navigate: Distilling Enterprise Knowledge into Navigable Agent Skills for QA and RAG
- URL: https://arxiv.org/abs/2604.14572
- 重要度: ⚡High
- 要約:
  1. [課題] 通常の RAG は取得済み文書の受動的消費に留まり、コーパス全体構造や未探索領域を踏まえた探索ができない。
  2. [手法] 文書群を階層的な skill directory にオフライン変換し、エージェントが要約ツリーを辿って枝を選びつつ必要文書へ到達する Corpus2Skill を提案した。
  3. [結果] enterprise QA ベンチマークで dense retrieval、RAPTOR、agentic RAG を全体的に上回った。
- 一言メモ: 社内文書・XBRL・決算資料を「埋め込み検索」だけでなく「ナビゲーション可能な知識構造」に変える設計は、そのまま金融 RAG の改善候補になる。

### [T4] How to Fine-Tune a Reasoning Model? A Teacher-Student Cooperation Framework to Synthesize Student-Consistent SFT Data
- URL: https://arxiv.org/abs/2604.14164
- 重要度: ⚡High
- 要約:
  1. [課題] 強い教師モデルの合成データで reasoning model を SFT すると、教師と生徒のスタイル差で性能が落ちることがある。
  2. [手法] 教師と生徒が交互にトークン生成を分担し、生徒分布に整合した合成データを作る TESSY を提案した。

- 一言メモ: 金融 SFT データ生成でも、強い教師の正解をそのまま使うより、目標モデルの文体・推論癖に合わせた蒸留設計が効きそうだ。

### [T4] C2: Scalable Rubric-Augmented Reward Modeling from Binary Preferences
- URL: https://arxiv.org/abs/2604.13618
- 重要度: 🔵Medium
- 要約:
  1. [課題] rubric つき reward modeling は有効だが、rubric 作成コストが高く、質の低い rubric は逆効果になりうる。
  2. [手法] binary preference だけから「有益な rubric」と「有害な rubric」を対照学習し、協調的 rubric generator と批判的 verifier を組み合わせる C2 を提案した。
  3. [結果] 外部 rubric 注釈なしで reward model の判断精度を改善し、大型モデル由来 rubric に近い性能を小型モデルで実現した。
- 一言メモ: 構造化出力の評価基準を rubric 化して reward に落とすと、単純な好み学習より実務的な出力品質制御に近づく。

### [T4] LongAct: Harnessing Intrinsic Activation Patterns for Long-Context Reinforcement Learning
- URL: https://arxiv.org/abs/2604.14922
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長文脈 RL では、全重みを一様更新すると重要な表現に学習が集中せず効率が悪い。

- 一言メモ: 長い決算資料や社内文書を扱う post-training で、LoRA の更新対象を賢く絞る発想に接続できる。

### [T1] KV Packet: Recomputation-Free Context-Independent KV Caching for LLMs
- URL: https://arxiv.org/abs/2604.13226
- 重要度: ⚡High
- 要約:
  1. [課題] 既存 KV cache は文脈依存なので、同じ文書を別コンテキストで再利用する際に再計算が必要になり TTFT が悪化する。
  2. [手法] 文書 KV を immutable packet として保持し、軽量 soft-token adapter で文脈断絶を補う再計算不要の再利用法を提案した。
  3. [結果] Llama-3.1 と Qwen2.5 でほぼゼロ FLOPs の追加で、再計算系ベースラインより低 TTFT かつ同等級の F1 を達成した。
- 一言メモ: 社内資料や頻出テンプレ文書を何度も参照する金融 RAG では、PCIe 環境の遅延削減に直結しうる。

- URL: https://arxiv.org/abs/2604.12988
- 重要度: ⚡High
- 要約:
  1. [課題] NL2SQL の Execution Accuracy は、構文差や曖昧な自然言語意図、誤った正解 SQL に弱く信頼しづらい。

### [T4] Self-Distillation Zero: Self-Revision Turns Binary Rewards into Dense Supervision
- URL: https://arxiv.org/abs/2604.12002
- 重要度: ⚡High
- 要約:
  1. [課題] verifiable task の RL は binary reward しかなく学習効率が低く、教師あり蒸留は外部教師や良質デモを必要とする。

### [T4] Target Policy Optimization
- URL: https://arxiv.org/abs/2604.06159
- 重要度: ⚡High
- 要約:
  1. [課題] policy-gradient 系手法は「どの出力に確率質量を寄せるか」と「そのためにどう更新するか」を同時に解くため、学習率や clipping に不安定性が残る。
  2. [手法] 報酬つき候補から target policy を先に構成し、その分布への cross-entropy 学習に分離する TPO を提案した。
  3. [結果] sparse reward 条件で PG、PPO、GRPO、DG を大きく上回り、LLM RLVR でも有効だった。
- 一言メモ: 構造化出力の合否が疎な報酬しか得られない場面では、GRPO 系より扱いやすい RL 更新則候補になる。

### [T3] Memory Transfer Learning: How Memories are Transferred Across Domains in Coding Agents
- URL: https://arxiv.org/abs/2604.14004
- 重要度: 🔵Medium
- 要約:
  1. [課題] エージェント memory は通常、同一ドメイン内だけで使われ、異種タスク間の再利用原理が整理されていない。
  2. [手法] 複数ベンチマークを横断する unified memory pool を用い、低レベル trace から高レベル insight まで転移性を比較した。
  3. [結果] 平均性能は改善し、特に高抽象度のメタ知識が転移しやすく、具体的すぎる痕跡は負の転移を起こしやすいと分かった。
- 一言メモ: 金融エージェント memory も、個別タスクのログ保存より「検証手順」や「失敗パターン」の抽象化が価値を持つ。

### [T4] From P(y|x) to P(y): Investigating Reinforcement Learning in Pre-train Space
- URL: https://arxiv.org/abs/2604.14142
- 重要度: ⚡High
- 要約:
  1. [課題] RLVR は条件付き分布 P(y|x) の最適化に留まり、元の事前分布が弱いと reasoning の伸びしろが頭打ちになる。
  2. [手法] 事前学習空間の P(y) を直接報酬駆動で更新する PreRL と、そこから通常 RL に移る Dual Space RL を提案した。
  3. [結果] 負例強化が reasoning 空間の剪定に強く効き、既存ベースラインを安定して上回った。
- 一言メモ: 金融ドメインの思考癖自体を事前分布側へ寄せたいなら、SFT 後の RL だけでなく pre-train space 的な更新発想も面白い。
