### [T1] Key-Value Means
- URL: https://arxiv.org/abs/2605.09877
- 重要度: ⚡High
- 要約:
  1. [課題] 長文推論では通常の attention が KV cache と prefill コストの両面で重く、長コンテキスト実用化の障害になっている。
  2. [手法] 固定長にも可変長にも対応する block-recurrent attention である KVM を導入し、標準演算だけで O(N) 近傍の chunked RNN と subquadratic prefill を実現する。
  3. [結果] 長文ベンチで競争力を保ちながら KV cache 削減と計算量制御を両立し、 custom kernel なしで実装可能であることを示した。
- 一言メモ: PCIe 環境での長文 RAG とエージェント履歴保持に直結するため、MoE 本体の前段や補助モデルで試す価値が高い。

### [T1] SlimSpec: Low-Rank Draft LM-Head for Accelerated Speculative Decoding
- URL: https://arxiv.org/abs/2605.10453
- 重要度: ⚡High
- 要約:
  1. [課題] speculative decoding では軽量 drafter でも巨大語彙への LM-head 射影が律速になりやすい。
  2. [手法] 語彙削減ではなく drafter の LM-head 内部表現を low-rank 化する SlimSpec を提案し、完全語彙を維持したまま計算を圧縮する。
  3. [結果] EAGLE-3 系で標準 LM-head 比 4〜5倍高速化し、既存法より end-to-end speedup を最大 8〜9% 改善した。
- 一言メモ: RTX 6000 Pro 群でのレイテンシ改善にそのまま効くため、AWQ/INT4 と併用する推論高速化候補として有力。

### [T1] Memory-Efficient Looped Transformer: Decoupling Compute from Memory in Looped Language Models
- URL: https://arxiv.org/abs/2605.07721
- 重要度: 🔵Medium
- 要約:
  1. [課題] recurrent/looped LLM は埋め込み空間で反復推論できる一方、反復ごとに KV cache が増えてメモリが線形増加する。
  2. [手法] 反復間で単一 KV cache を共有し gating で更新する MELT を導入し、二段階の post-training で既存 LoopLM から移行する。
  3. [結果] reasoning depth とメモリ消費を切り離し、標準 LLM 並みのメモリで LoopLM 系の性能を維持した。
- 一言メモ: PCIe 前提で test-time compute を積みたいときの現実解で、長い検証ループを持つ金融エージェントと相性がよい。

### [T1] DECO: Sparse Mixture-of-Experts with Dense-Comparable Performance on End-Side Devices
- URL: https://arxiv.org/abs/2605.10933
- 重要度: 🔵Medium
- 要約:
  1. [課題] MoE は計算効率が良くても総パラメータ量とメモリアクセスが重く、端末側や省メモリ運用では不利になりやすい。
  2. [手法] ReLU ベース routing と expert-wise scaling、NormSiLU を組み合わせた高疎性 MoE の DECO を提案する。
  3. [結果] 20% の expert 活性化で dense Transformer 同等性能を達成し、専用 kernel で dense 推論比 3倍高速化した。
- 一言メモ: NVLink なしの Expert Parallelism を前提にするなら、 routing 単純化と実効疎性向上の設計はそのまま参考になる。

### [T2/T3] Rethinking Agentic Search with Pi-Serini: Is Lexical Retrieval Sufficient?
- URL: https://arxiv.org/abs/2605.10848
- 重要度: ⚡High
- 要約:
  1. [課題] deep research 系エージェントで dense retriever が本当に必須かは再検証の余地がある。
  2. [手法] BM25 ベースの lexical retrieval に retrieving・browsing・reading の 3 ツールを備えた Pi-Serini を構築し、 retrieval depth と BM25 設定を体系比較した。
  3. [結果] BrowseComp-Plus で gpt-5.5 と組み合わせて 83.1% accuracy、94.7% surfaced evidence recall を達成し、 dense retriever 系公開エージェントを上回った。
- 一言メモ: 社内文書や XBRL 検索でも、まず lexical の深掘り最適化で十分戦える可能性があり、RAG 投資順序を見直せる。

### [T3] Conformal Agent Error Attribution
- URL: https://arxiv.org/abs/2605.06788
- 重要度: 🔵Medium
- 要約:

### [T4] Prompt-Activation Duality: Improving Activation Steering via Attention-Level Interventions
- URL: https://arxiv.org/abs/2605.10664

- 要約:
  1. [課題] activation steering は対話が長くなると KV-cache 汚染で一貫性が崩れやすい。
