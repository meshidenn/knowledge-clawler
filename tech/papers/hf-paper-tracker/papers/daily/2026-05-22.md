### [T1] OScaR: The Occam's Razor for Extreme KV Cache Quantization in LLMs and Beyond
- URL: https://arxiv.org/abs/2605.19660
- 重要度: ⚡High
- 要約:
  1. [課題] 長文推論とマルチモーダル化でKV cacheが主要なメモリ律速になり、既存のper-channel量子化は極端な低bit化で精度を保ちにくい。
  2. [手法] 誤差の主因をToken Norm Imbalanceとして捉え、Canalized RotationとOmni-Token Scalingを組み合わせた軽量なINT2向けKV圧縮フレームワークOScaRを提案。
  3. [結果] 複数のX-LLMで既存法を上回る新しいPareto frontを示し、BF16比で最大3.0倍デコード高速化、5.3倍メモリ削減、4.1倍スループット向上を達成した。
- 一言メモ: PCIe環境で長文RAGとエージェント推論を回す自分たちの条件に最も直結する、今日の本命T1論文。

### [T1][T3] Mix-Quant: Quantized Prefilling, Precise Decoding for Agentic LLMs
- URL: https://arxiv.org/abs/2605.20315
- 重要度: ⚡High
- 要約:
  1. [課題] エージェント型LLMは長い履歴やツール出力を毎回prefillするため、decodeよりprefillが実運用ボトルネックになりやすい。
  2. [手法] 精度劣化が大きい全段量子化を避け、prefillだけをNVFP4で量子化しdecodeはBF16のまま保つphase-aware quantizationであるMix-Quantを提案。
  3. [結果] 長文・agenticベンチで性能をほぼ維持したまま、prefillingを最大3倍高速化した。
- 一言メモ: マルチターンのMCP実行や長い社内文書投入で重いのはまさにprefillなので、量子化の当てどころとしてかなり実務的。

### [T1] TIDE: Efficient and Lossless MoE Diffusion LLM Inference with I/O-aware Expert Offload
- URL: https://arxiv.org/abs/2605.20179
- 重要度: 🔵Medium
- 要約:
  1. [課題] MoE化したdiffusion LLMはGPU外へのexpert offload時にI/O負荷が大きく、資源制約下での推論効率が落ちやすい。
  2. [手法] diffusion block内でexpert活性が時間的に安定する性質を使い、I/Oを意識したinterval-based expert refreshと最適スケジューリングを行うTIDEを提案。
  3. [結果] 学習不要のlossless最適化として、単一GPU-CPU環境でLLaDA系モデルのスループットを最大1.4〜1.5倍改善した。
- 一言メモ: ベースモデルはAR前提でも、expert配置更新頻度をI/O制約込みで最適化する発想はPCIe前提のMoE運用に転用余地がある。

### [T2] Toto 2.0: Time Series Forecasting Enters the Scaling Era
- URL: https://arxiv.org/abs/2605.20119
- 重要度: 🔵Medium
- 要約:
  1. [課題] 時系列基盤モデルは本当にスケール則に従うのか、また単一レシピで安定に性能向上できるのかが未整理だった。
  2. [手法] 4Mから2.5Bまで同一訓練レシピで拡張した5種のオープン時系列基盤モデルToto 2.0を構築し、u-muPを含む設計判断を体系化した。
  3. [結果] BOOM、GIFT-Eval、TIMEの3ベンチでSOTAを更新し、時系列基盤モデルでも明確なスケーリング利益があることを示した。
- 一言メモ: 市場データやポートフォリオ時系列を扱う補助モジュールとして、LLM本体とは別に予測専用foundation modelを持つ設計が現実味を増している。

### [T3] Evaluating Temporal Semantic Caching and Workflow Optimization in Agentic Plan-Execute Pipelines
- URL: https://arxiv.org/abs/2605.20630
- 重要度: ⚡High
- 要約:
  1. [課題] plan-execute型エージェントでは、tool discovery、LLM planning、MCP実行、要約が繰り返され、通常のKV再利用や埋め込みキャッシュだけでは正当性を保てない。
  2. [手法] 時間・資産・センサ条件に依存する問い合わせを前提に、temporal semantic cacheとMCP向けworkflow最適化を組み合わせて評価した。
  3. [結果] workflow最適化で1.67倍高速化、中央値レイテンシ約40%削減、temporal cacheのヒット時は中央値30.6倍高速化を示した。
- 一言メモ: MCP連携を前提にした金融エージェントそのものの性能課題に近く、単なるモデル改善より先に効く実装項目が多い。

### [T3] SaaSBench: Exploring the Boundaries of Coding Agents in Long-Horizon Enterprise SaaS Engineering
- URL: https://arxiv.org/abs/2605.17526
- 重要度: ⚡High
- 要約:
  1. [課題] 既存のcoding agentベンチは単純化された単一スタックに偏り、実際の企業システムにある多言語・多DB・多コンポーネント統合の難しさを測れていない。
  2. [手法] 8言語、6DB、13フレームワークを含む30タスクから成るSaaSBenchと、依存関係を考慮したhybrid評価方式を構築した。
  3. [結果] 95%以上の失敗が深い業務ロジック以前の統合・設定段階で発生し、現状エージェントの主ボトルネックがシステム統合だと示した。
- 一言メモ: 金融AIエージェントでも難所は推論品質だけでなく、外部ツール・データ・権限系の統合だと再確認できる。

### [T3] MINTEval: Evaluating Memory under Multi-Target Interference in Long-Horizon Agent Systems
- URL: https://arxiv.org/abs/2605.18565
- 重要度: ⚡High
- 要約:
  1. [課題] 長期エージェントでは更新される多数の情報が互いに干渉するが、既存評価は静的で独立な記憶再生に偏っていた。
  2. [手法] 状態追跡、対話、Wikipedia改訂、GitHub commitなどを含む最大180万token級の干渉多発ベンチMINTEvalを構築した。
  3. [結果] vanilla long-context、RAG、memory-augmentedを含む7系統すべてで平均精度27.9%と低く、特に複数証拠の統合推論で大きく崩れることを示した。
- 一言メモ: 社内RAGや継続対話の記憶層は「保存量」より更新干渉への耐性が重要で、現行設計の弱点を測るベンチとして有用。

### [T3] S-Bus: Automatic Read-Set Reconstruction for Multi-Agent LLM State Coordination
- URL: https://arxiv.org/abs/2605.17076
- 重要度: 🔵Medium
- 要約:
  1. [課題] 複数エージェントが自然言語状態を共有すると、stale-readやwrite競合によるStructural Race Conditionが静かに出力を壊しうる。
  2. [手法] HTTPトラフィックから各agentのread setを自動再構築し、Observable-Read Isolationで整合性を守るS-Bus middlewareを提案した。
  3. [結果] TLAPS・TLC・Dafnyによる機械検証と、PostgreSQL/Redis比較実験で多数競合下でもType-I corruptionゼロを示した。
- 一言メモ: マルチエージェントの共有メモリやタスク状態管理で、prompt工夫ではなく整合性制御層を置く必要性を示している。

### [T4] You Only Need Minimal RLVR Training: Extrapolating LLMs via Rank-1 Trajectories
- URL: https://arxiv.org/abs/2605.21468
- 重要度: ⚡High
- 要約:
  1. [課題] RLVRは効果的だが学習コストが高く、どこまで短い観測から将来checkpointを予測できるかが不明だった。
  2. [手法] RLVRの重み更新がほぼrank-1軌跡に乗ることを見出し、短い観測窓から線形外挿するRELEXを提案した。
  3. [結果] Qwen2.5-Math-1.5BやQwen3系でフルRLVRの15%程度の学習ステップから同等以上の性能を再現し、50 step観測から1000 step先まで外挿できた。
- 一言メモ: DPO/KTOや構造化出力改善の試行回数を大きく減らせる可能性があり、限られたGPU予算での反復速度に効く。

### [T4] Learning from Language Feedback via Variational Policy Distillation
- URL: https://arxiv.org/abs/2605.15113
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLVRは結果報酬が疎で探索が難しく、既存の言語フィードバック蒸留も固定teacherの解釈能力が頭打ちになりやすい。
  2. [手法] teacherとstudentを同時進化させるVariational EMとして定式化し、teacher側も軌跡結果に応じて適応更新するVPDを提案。
  3. [結果] 科学推論やコード生成で標準RLVRおよび既存自己蒸留法を一貫して上回り、言語批評をより学習可能な密報酬へ変換した。
- 一言メモ: 構造化出力の採点コメントや監査コメントを、固定judgeではなく共進化型teacherとして使う設計に近い。

### [T4] Safety Alignment as Continual Learning: Mitigating the Alignment Tax via Orthogonal Gradient Projection
- URL: https://arxiv.org/abs/2602.07892
- 重要度: 🔵Medium
- 要約:
  1. [課題] 安全アライメントは有用性低下を招くalignment taxを伴いやすく、SFT→DPOの逐次学習で既存能力を壊しやすい。
  2. [手法] 汎用能力データ上の勾配部分空間を推定し、その方向成分を安全勾配から除去するOrthogonal Gradient Projectionを提案。
  3. [結果] Qwen2.5-7B-InstructやLlama3.1-8B-Instructで、標準手法より安全性と有用性のトレードオフを改善した。
- 一言メモ: 金融コンプライアンス強化で汎用分析力を落としたくない自分たちにとって、逐次アライメントの勾配干渉を抑える視点は重要。

### [T4] Not Every Rubric Teaches Equally: Policy-Aware Rubric Rewards for RLVR
- URL: https://arxiv.org/abs/2605.20164
- 重要度: ⚡High
- 要約:
  1. [課題] rubric-based rewardは人間が重視する項目と、現在の方策にとって学習信号として有効な項目を混同しがちで、RL効率を落とす。
  2. [手法] 最終評価のrubric重みは維持しつつ、rollout間の識別力に応じて学習時の項目重みを動的調整するPOW3Rを提案。
  3. [結果] 3つの基礎方策・2データセットで30比較中24勝し、strict completionを改善しながら2.5〜4倍少ないステップで同等水準へ到達した。
- 一言メモ: JSON schema準拠、根拠記載、コンプライアンス表現など複数基準を持つ金融出力の学習設計にそのまま応用しやすい。

### [T4] The Unlearnability Phenomenon in RLVR for Language Models
- URL: https://arxiv.org/abs/2605.16787
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLVRでは正解軌跡が存在しても学習できないhard exampleがあり、その原因が十分理解されていなかった。
  2. [手法] 交差サンプル勾配解析を用いて、unlearnable例が他例と勾配類似性の低い表現欠陥を持つことを解析した。
  3. [結果] 最適化手法やsampling改善、data augmentationではこの問題が解けず、現行RLVRの根本限界を示した。
- 一言メモ: SFTデータ品質が最重要課題である自分たちにとって、報酬設計以前に「学べる表現の土台」を整える必要があると分かる。
