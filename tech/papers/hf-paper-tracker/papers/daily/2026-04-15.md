### [T2/T3] Time is Not a Label: Continuous Phase Rotation for Temporal Knowledge Graphs and Agentic Memory
- URL: https://arxiv.org/abs/2604.11544
- 重要度: ⚡High
- 要約:
  1. [課題] エージェントの長期記憶では、時間変化する事実と永続的な事実を同じように扱うため、古いが有効な知識の埋没や誤上書きが起きやすい。
  2. [手法] 関係ごとの変動性を学習する Semantic Speed Gate と continuous phase rotation を組み合わせ、古い事実を削除せず位相的に後景化する RoMem を提案。
  3. [結果] ICEWS05-15 で SOTA の 72.6 MRR を達成し、agentic memory 評価でも 2〜3倍の MRR・回答精度改善、金融ドメインへのゼロショット一般化も示した。
- 一言メモ: 社内文書や市場イベントの時系列知識を RAG だけでなく構造化メモリとして持つ設計に直結する。

### [T1] IceCache: Memory-efficient KV-cache Management for Long-Sequence LLMs
- URL: https://arxiv.org/abs/2604.10539
- 重要度: ⚡High
- 要約:
  1. [課題] 長文推論時の KV cache はメモリを線形に消費し、GPU 制約下では CPU オフロードも精度とレイテンシを悪化させやすい。
  2. [手法] セマンティックなトークンクラスタリングを PagedAttention に統合し、CPU-GPU 転送時に重要トークンを効率よく保持する IceCache を提案。
  3. [結果] LongBench で 256-token budget でもフル KV cache の 99% 精度を維持し、他手法より 25% の token budget で競争力ある精度と速度を示した。
- 一言メモ: PCIe 接続の RTX 6000 Pro 前提で長文金融文書を回す現実的な推論最適化候補として優先度が高い。

### [T3/T4] TRACE: Capability-Targeted Agentic Training
- URL: https://arxiv.org/abs/2604.05336
- 重要度: ⚡High
- 要約:
  1. [課題] エージェントの失敗は環境ごとに不足能力が異なるため、汎用的な合成データや単純な RL では改善点がぼやける。
  2. [手法] 成功・失敗軌跡の差分から不足 capability を抽出し、その能力だけを発火させる合成環境を作って LoRA adapter を RL 学習し、推論時にルーティングする TRACE を提案。
  3. [結果] τ^2-bench で +14.1pt、ToolSandbox で +7 perfect score を達成し、GRPO や GEPA より少ないロールアウトで高効率に改善した。
- 一言メモ: 金融 AI エージェントでも「ツール選択」「検証」「構造化出力」など能力別 LoRA を分ける設計の雛形になる。

### [T3] Agentic Aggregation for Parallel Scaling of Long-Horizon Agentic Tasks
- URL: https://arxiv.org/abs/2604.11753
- 重要度: ⚡High
- 要約:
  1. [課題] Deep research のような長期タスクでは並列ロールアウトを増やしても、最終回答だけの多数決では途中軌跡の有益情報を活かせない。
  2. [手法] 複数軌跡を探索対象とみなし、候補解や中間情報を必要時に検索・統合する aggregation agent の AggAgent を提案。
  3. [結果] 6ベンチマーク・3モデル群で既存集約法を平均最大 5.3% 上回り、deep research 系では 10.3% の改善を示しつつ、追加コストは単一ロールアウト相当に抑えた。
- 一言メモ: 複数エージェントで調査した市場分析結果を統合する最終アグリゲータ設計にそのまま転用しやすい。

### [T4] Efficient RL Training for LLMs with Experience Replay
- URL: https://arxiv.org/abs/2604.08706
- 重要度: ⚡High
- 要約:
  1. [課題] LLM の post-training では fresh な on-policy データ偏重が強く、生成コストが高い環境では学習効率が悪い。
  2. [手法] リプレイバッファ設計を、staleness による分散増加・多様性・生成コストのトレードオフとして定式化し、再利用戦略を体系的に検証した。
  3. [結果] 適切な replay buffer を使うと推論計算量を大きく削減しつつ性能を維持し、場合によっては最終性能も改善できることを示した。
- 一言メモ: 高価な金融タスク軌跡を使う DPO/KTO/RL で、ロールアウト再利用の実務設計を見直す根拠になる。

### [T4] The Past Is Not Past: Memory-Enhanced Dynamic Reward Shaping
- URL: https://arxiv.org/abs/2604.11297
- 重要度: ⚡High
- 要約:
  1. [課題] RL 後の LLM は同じ誤りパターンを繰り返しやすく、通常のエントロピー正則化だけでは失敗多様性を十分に制御できない。
  2. [手法] 過去ロールアウトの中間表現を保存し、密度ベースのクラスタリングで頻出失敗を検出して、そのクラスターに追加ペナルティを与える MEDS を提案。
  3. [結果] 5データセット・3モデルで一貫して既存法を上回り、最大で pass@1 を 4.13pt、pass@128 を 4.37pt 改善し、行動多様性向上も確認した。
- 一言メモ: 構造化出力の壊れ方が似通う問題に対し、失敗クラスタ単位で報酬を入れる設計が有効そう。

### [T4] SCOPE: Signal-Calibrated On-Policy Distillation Enhancement with Dual-Path Adaptive Weighting
- URL: https://arxiv.org/abs/2604.10688
- 重要度: 🔵Medium
- 要約:
  1. [課題] On-policy RL は疎な outcome reward に依存するため、誤軌跡と正軌跡を同じ重みで蒸留すると token-level credit assignment が粗くなる。
  2. [手法] 誤答軌跡には teacher-perplexity 重み付き KL、正答軌跡には student-perplexity 重み付き MLE を適用する二経路の適応重み付けフレームワーク SCOPE を提案。
  3. [結果] 6つの reasoning benchmark で Avg@32 を平均 11.42%、Pass@32 を 7.30% 改善し、安定した性能向上を示した。
- 一言メモ: 正答でも「境界事例だけ強化する」発想は、金融用の schema 違反修正学習に相性がよい。

### [T3/T4] Mobile GUI Agent Privacy Personalization with Trajectory Induced Preference Optimization
- URL: https://arxiv.org/abs/2604.11259
- 重要度: 🔵Medium
- 要約:
  1. [課題] GUI エージェントの個人化では、プライバシー志向と効率志向で軌跡構造自体が変わるため、通常の preference optimization が不安定になる。
  2. [手法] 重要なプライバシー関連ステップを強調し、padding gating でノイズを抑える Trajectory Induced Preference Optimization（TIPO）を提案。
  3. [結果] Privacy Preference Dataset で 65.60% SR、46.22 Compliance、66.67% PD を達成し、既存最適化法を上回った。
- 一言メモ: 金融エージェントでも「積極提案型」と「保守・コンプラ重視型」を軌跡ベースで分けて学習できる。

### [T1] SPEED-Bench: A Unified and Diverse Benchmark for Speculative Decoding
- URL: https://arxiv.org/abs/2604.09557
- 重要度: 🔵Medium
- 要約:
  1. [課題] Speculative Decoding はデータ依存性が強いのに、既存ベンチは負荷多様性や本番推論エンジンでの評価が不足している。
  2. [手法] semantic diversity を重視した Qualitative split と、同時実行数を振れる Throughput split を持つ SPEED-Bench を構築し、vLLM と TensorRT-LLM に統合した。
  3. [結果] 合成入力が実運用 throughput を過大評価することや、バッチサイズごとの最適 draft length、vocabulary pruning の罠を可視化した。
- 一言メモ: Qwen3.5-397B-A17B 級 MoE を本番運用する前に、投資メモ・XBRL・長文 QA で専用 workload を組む必要性を示している。

### [T3] CocoaBench: Evaluating Unified Digital Agents in the Wild
- URL: https://arxiv.org/abs/2604.11201
- 重要度: ⚡High
- 要約:
  1. [課題] 既存エージェント評価は SWE、GUI、検索などを個別に測るものが多く、複合能力を要する実運用タスクを捉えにくい。
  2. [手法] 視覚・検索・コーディングを柔軟に組み合わせる長期タスクを、人手設計の instruction と自動評価関数で定義した CocoaBench と共通 scaffold の CocoaAgent を提案。
  3. [結果] 最良システムでも成功率は 45.1% に留まり、推論計画・ツール実行・視覚グラウンディングの改善余地が大きいことを示した。
- 一言メモ: 金融業務フローを模した統合ベンチを作る際の設計見本として有用で、単機能ベンチ卒業のタイミングを示す。

### [T3] CodeTracer: Towards Traceable Agent States
- URL: https://arxiv.org/abs/2604.11641
- 重要度: 🔵Medium
- 要約:
  1. [課題] コードエージェントは並列ツール呼び出しや多段ワークフローで内部状態遷移が見えにくく、失敗原因の特定が難しい。
  2. [手法] 異種ラン成果物から階層的 trace tree を復元し、失敗起点と伝播連鎖を特定する CodeTracer と CodeTraceBench を提案。
  3. [結果] 既存の直接 prompting や軽量ベースラインより高精度に失敗箇所を局在化し、診断信号を再投入すると失敗 run の回復も改善した。
- 一言メモ: 金融 AI エージェントの監査性と障害解析を強化するため、trace schema 設計の優先度が上がる。

### [T3/T4] Tracing the Roots: A Multi-Agent Framework for Uncovering Data Lineage in Post-Training LLMs
- URL: https://arxiv.org/abs/2604.10480
- 重要度: 🔵Medium
- 要約:
  1. [課題] post-training 用データセットは系譜関係が見えず、重複・汚染・多様性不足が見逃されやすい。
  2. [手法] 複数エージェントでデータ lineage graph を再構成し、上流ソース起点で instruction sampling する lineage-aware 多様化手法を提案。
  3. [結果] 数学系の縦方向 refinement や汎用データの横方向 aggregation などの構造パターンを可視化し、benchmark contamination と hidden redundancy も発見した。
- 一言メモ: 金融 SFT データの品質改善で悩んでいる現状に対し、データ系譜管理を入れる価値が大きい。

### [T2/T4] Polyglot Teachers: Evaluating Language Models for Multilingual Synthetic Data Generation
- URL: https://arxiv.org/abs/2604.11290
- 重要度: 🔵Medium
- 要約:
  1. [課題] 多言語 SFT 合成データでは、単に大きい teacher を選ぶと非英語で品質が落ち、student 性能も頭打ちになりやすい。
  2. [手法] 10モデル・6言語・140万件超の合成データ・240 student を用い、内在品質と下流性能を束ねた Polyglot Score で teacher 選定要因を分析した。
  3. [結果] Gemma 3 27B と Aya Expanse 32B が安定した teacher となり、モデルサイズよりも prompt diversity・長さ・流暢さが性能を強く予測した。
- 一言メモ: 多言語決算資料や海外 IR 向けの SFT では、teacher 選定をサイズ主義からデータ品質主義へ切り替えるべきだと分かる。

### [T3] ATANT: An Evaluation Framework for AI Continuity
- URL: https://arxiv.org/abs/2604.06710
- 重要度: 🔵Medium
- 要約:
  1. [課題] RAG や profile layer などの memory 機構は増えているが、時間をまたいだ一貫性を正式に測る枠組みが整っていない。
  2. [手法] continuity を 7特性で定義し、LLM を評価ループに入れず 10 checkpoints で検証する ATANT と 250 ストーリーの narrative corpus を提案。
  3. [結果] 参照実装は 250 ストーリー累積条件でも 96% に到達し、文脈混線なく正しい事実を取り出せるかを測る評価法として機能した。
- 一言メモ: 顧客ごとの投資方針や案件履歴を扱うエージェントで、メモリ汚染を防ぐ評価基盤として参考になる。
