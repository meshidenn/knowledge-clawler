### [T1/T4] MASCing: Configurable Mixture-of-Experts Behavior via Activation Steering Masks
- URL: https://arxiv.org/abs/2604.27818
- 重要度: ⚡High
- 要約:
  1. [課題] MoEは疎活性化で推論効率を得る一方、ルーティング依存で安全性や振る舞い制御が難しい。
  2. [手法] ルーティングlogitから挙動を予測する代理モデルを学習し、推論時に steering mask で専門家選択を上書きする軽量制御法を提案した。
  3. [結果] 7つの公開MoEで追加コストほぼ無しのまま、多ターン jailbreak 防御成功率や挙動切替性能を大きく改善した。
- 一言メモ: 金融向けMoE基盤で、安全制約や出力方針を再学習なしに切り替える実装候補としてかなり現実的。

### [T4/T2] How Fast Should a Model Commit to Supervision? Training Reasoning Models on the Tsallis Loss Continuum
- URL: https://arxiv.org/abs/2604.25907
- 重要度: ⚡High
- 要約:
  1. [課題] 出力レベル報酬だけで reasoning model を新規タスクへ適応させると、初期成功率が低い条件で RLVR が cold start から抜け出せない。
  2. [手法] RLVR と潜在軌跡の尤度最大化を連続的につなぐ Tsallis 損失族を導入し、GARL と PAFT という2つの推定器で実用化した。
  3. [結果] FinQA・HotPotQA・MuSiQueで cold-start 停滞を大幅に緩和し、HotPotQA では GRPO 比で大きな改善を示した。
- 一言メモ: 金融 QA や構造化推論の後学習で、報酬疎薄な初期段階を抜けるための RL/SFT 中間設計として直接試す価値がある。

### [T2/T3] Chain of Evidence: Pixel-Level Visual Attribution for Iterative Retrieval-Augmented Generation
- URL: https://arxiv.org/abs/2605.01284
- 重要度: ⚡High
- 要約:
  1. [課題] 既存 iRAG はテキスト化した文書に依存するため、図表やレイアウトを含むPDFで根拠追跡と視覚的意味が失われやすい。
  2. [手法] 取得文書のスクリーンショットをVLMで直接読ませ、根拠の bbox を含む reasoning chain を返す retriever-agnostic な visual attribution 枠組みを提案した。
  3. [結果] SlideVQA など視覚レイアウト理解が必要な設定で、テキストベース手法を大きく上回る性能を示した。
- 一言メモ: 決算説明資料やIRスライドの図表根拠をそのまま示せるので、金融RAGの説明可能性と監査性を一段上げられる。

### [T2/T3] Hierarchical Abstract Tree for Cross-Document Retrieval-Augmented Generation
- URL: https://arxiv.org/abs/2605.00529
- 重要度: ⚡High
- 要約:
  1. [課題] 既存 Tree-RAG は単一文書前提が強く、複数文書に跨る multi-hop 質問で構造分離や粗い抽象化がボトルネックになる。
  2. [手法] 分布適応的に構築する hierarchical abstract tree と、再構成クエリを用いる multi-granular retrieval agent を組み合わせた Ψ-RAG を提案した。
  3. [結果] cross-document multi-hop QA で RAPTOR や HippoRAG 2 を大きく上回る平均F1を達成した。
- 一言メモ: 社内文書・XBRL・決算資料を横断する金融RAGでは、単純 chunk 検索より階層索引と探索エージェントの併用が効きそう。

### [T2] Prior-Aligned Data Cleaning for Tabular Foundation Models
- URL: https://arxiv.org/abs/2604.25154
- 重要度: 🔵Medium
- 要約:
  1. [課題] Tabular Foundation Model は欠損・外れ値・重複を含む実データで prior mismatch を起こし、精度と校正が同時に悪化する。
  2. [手法] クリーニング操作列を RL で最適化し、汚れた入力をモデルの事前分布へ近づける prior alignment として定式化した。
  3. [結果] 10データセットで静的前処理より良い cleaning pipeline を見つけ、他データセットへの転移も確認した。
- 一言メモ: 財務テーブルや市場データの前処理を固定ルールではなくモデル前提に合わせて最適化する発想として参考になる。

### [T3] ARIS: Autonomous Research via Adversarial Multi-Agent Collaboration
- URL: https://arxiv.org/abs/2605.03042
- 重要度: ⚡High
- 要約:
  1. [課題] 長時間動く研究エージェントは、もっともらしいが根拠不十分な成果を静かに積み上げる failure mode を持つ。
  2. [手法] 実行役と別系統モデルのレビュー役を常設し、skills・MCP・永続 wiki・claim auditing を束ねた adversarial multi-agent harness を構築した。
  3. [結果] 65超の再利用スキル、5種類の end-to-end workflow、証拠と主張を突合する assurance layer を備えた実運用ハーネスとして提示した。
- 一言メモ: 金融AIエージェントでも、実行モデルと監査モデルを分離し、根拠台帳ベースで主張検証する設計が重要になる。

### [T3/T4] Reinforcement Learning for LLM-based Multi-Agent Systems through Orchestration Traces
- URL: https://arxiv.org/abs/2605.02801
- 重要度: ⚡High
- 要約:
  1. [課題] マルチエージェントのRLでは、個別アクションだけでなく spawn・delegate・aggregate・stop の最適化が整理されていない。
  2. [手法] オーケストレーション痕跡を時間グラフとして捉え、報酬設計・credit 単位・5つの下位意思決定に分解して研究地図を与えた。
  3. [結果] 84本の論文プールと replayable JSON schema を整備し、特に stopping decision を直接学習する手法が未開拓だと示した。
- 一言メモ: 自社のマルチエージェント基盤でも、停止判断や委譲品質を学習対象として切り出すべきだと分かる。

### [T4] Beyond SFT-to-RL: Pre-alignment via Black-Box On-Policy Distillation for Multimodal RL
- URL: https://arxiv.org/abs/2604.28123
- 重要度: 🔵Medium
- 要約:
  1. [課題] 標準的な SFT→RLVR パイプラインは supervision 分布からの drift を生み、知覚エラーと推論エラーを増幅しやすい。
  2. [手法] SFT と RL の間に、知覚 expert と reasoning expert を持つ MoE 判別器で応答分布を補正する on-policy distillation 段を挿入した。
  3. [結果] Qwen3-VL の 4B/8B で複数RLアルゴリズムに対して平均精度を一貫して改善した。
- 一言メモ: 直接は VLM 向けだが、金融向け SFT 後の distribution drift を抑える中間整列段の発想はテキスト LLM にも応用可能。

### [T3] OpenSeeker-v2: Pushing the Limits of Search Agents with Informative and High-Difficulty Trajectories
- URL: https://arxiv.org/abs/2605.04036
- 重要度: ⚡High
- 要約:
  1. [課題] 高性能 search agent の育成は、CPT・SFT・RL を含む重い産業レシピに依存し、再現性が低い。
  2. [手法] 知識グラフ拡張・ツール数拡大・低品質短手順の厳格除外という3つのデータ合成改良で、高難度 trajectory を作り込んだ。
  3. [結果] わずか 10.6k データの SFT だけで BrowseComp など4ベンチで同規模帯の SOTA を達成した。
- 一言メモ: 金融リサーチエージェントでも、RLを急ぐ前に「高難度で情報量の高い探索軌跡」を集める方が費用対効果が高そう。

- URL: https://arxiv.org/abs/2605.03596
- 重要度: 🔵Medium
- 要約:
  1. [課題] 既存ベンチは限定的なファイル集合に依存しており、実務のような大規模 workspace 依存を十分に評価できていない。
  2. [手法] 74種・2万超ファイルからなる現実的 workspace と、依存グラフ付き388タスクを含む評価基盤を構築した。

- 一言メモ: 社内文書・表計算・メモ・コードが混在する金融業務環境で、エージェント評価を組む際の直接的な参考例になる。

### [T3/T4] T^2PO: Uncertainty-Guided Exploration Control for Stable Multi-Turn Agentic Reinforcement Learning
- URL: https://arxiv.org/abs/2605.02178
- 重要度: ⚡High
- 要約:
  1. [課題] マルチターンRLでは、進捗に寄与しない低情報アクションが蓄積し、学習不安定化や崩壊を招く。
  2. [手法] 不確実性変化を監視して token 単位で thinking 介入し、turn 単位では探索進捗の薄い対話を再サンプルする T^2PO を提案した。
  3. [結果] WebShop・ALFWorld・Search QA で学習安定性と性能を改善し、探索効率の向上を示した。
- 一言メモ: 金融エージェント学習でも、「考える/進む/やり直す」を不確実性で制御する設計が有望。

### [T3] From Context to Skills: Can Language Models Learn from Context Skillfully?
- URL: https://arxiv.org/abs/2604.27660
- 重要度: ⚡High
- 要約:
  1. [課題] 長く専門的な文脈から必要な規則や手順を学ぶには、手動スキル化のコストが高く、自動抽出にも外部フィードバックが乏しい。
  2. [手法] Challenger・Reasoner・Judge を中心とする自己進化型 self-play で、文脈依存スキルを自律発見・精錬・選別する Ctx2Skill を提案した。
  3. [結果] CL-bench の4タスクで、抽出スキルを任意のLMへ差し込むだけで context learning 性能を一貫して改善した。
- 一言メモ: 規制文書や社内運用ルールから金融エージェント用スキルを自動抽出する方向にかなり近い。

### [T3/T4] HiL-Bench (Human-in-Loop Benchmark): Do Agents Know When to Ask for Help?
- URL: https://arxiv.org/abs/2604.09408
- 重要度: ⚡High
- 要約:
  1. [課題] 既存ベンチは曖昧さや情報欠落をほぼ含まず、エージェントが「いつ人に聞くべきか」を測れていない。
  2. [手法] human-validated blocker を含むタスクと、質問精度と blocker recall を両立させる Ask-F1 指標を導入した。
  3. [結果] フロンティアモデルでも selective escalation 能力は大きく不足しつつ、形状化報酬によるRLで改善可能だと示した。
- 一言メモ: 金融実務では勝手な補完が高コスト事故になるので、確認に戻る能力を明示的に評価・学習すべき。

### [T3/T4] Hallucinations Undermine Trust; Metacognition is a Way Forward
- URL: https://arxiv.org/abs/2605.01428
- 重要度: ⚡High
- 要約:
  1. [課題] 幻覚は知識不足だけでなく、自分の知識境界を認識して適切に不確実性表明できないことが本質的な問題である。
  2. [手法] 幻覚を「自信を伴う誤り」と再定義し、faithful uncertainty を中核とする metacognition の観点で整理した。
  3. [結果] 直接応答では不確実性表明、エージェントでは「いつ検索し何を信じるか」を司る制御層が重要だと論じた。
- 一言メモ: 構造化出力の品質改善でも、正答率だけでなく不確実性をどこで明示するかが実運用上の鍵になる。
