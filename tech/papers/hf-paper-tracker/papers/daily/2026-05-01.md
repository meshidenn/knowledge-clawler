### [T3] FAMA: Failure-Aware Meta-Agentic Framework for Open-Source LLMs in Interactive Tool Use Environments
- URL: https://arxiv.org/abs/2604.25135
- 重要度: 🔵Medium
- 要約:
  1. [課題] 小型オープンLLMのツール利用エージェントは、マルチターン対話で誤判断が連鎖しやすい。
  2. [手法] 失敗軌跡を分析して主要エラーを抽出し、その失敗型ごとに最小限の専門エージェントを起動して事前文脈を注入する二段構成を提案。
  3. [結果] 標準ベースライン比で評価設定全体にわたり最大27%の性能改善を示した。
- 一言メモ: 失敗類型ごとの補助エージェント起動は、金融AIエージェントの誤発注防止や根拠不足時の再確認フロー設計に直結する。

### [T2][T3] Operating-Layer Controls for Onchain Language-Model Agents Under Real Capital
- URL: https://arxiv.org/abs/2604.26091
- 重要度: ⚡High
- 要約:
  1. [課題] 実資金を扱うエージェントでは、モデル性能だけではなく実行前後の制御層まで含めた信頼性設計が必要になる。
  2. [手法] 21日間・3505体の実運用トレースをもとに、構造化コントロール、ポリシー検証、実行ガード、メモリ設計、観測性を含む operating layer を分析した。
  3. [結果] fabricated sell rules を57%から3%へ、fee-led observations を32.5%未満から10%未満へ抑え、資本投入率を42.9%から78.0%へ改善した。
- 一言メモ: 金融エージェントの本命論文で、構造化出力制約・実行前バリデーション・監査トレースを一体で設計すべきことを強く示している。

### [番外] Large Language Models Explore by Latent Distilling
- URL: https://arxiv.org/abs/2604.24927
- 重要度: 🔵Medium
- 要約:
  1. [課題] test-time scaling ではサンプリング多様化を増やしても意味的探索が浅く、Pass@k効率が伸びにくい。
  2. [手法] 浅層表現から深層表現を予測する軽量 Distiller を推論時に適応学習し、予測誤差を novelty signal として使う Exploratory Sampling を提案。
  3. [結果] 推論オーバーヘッド5%未満で、数学・科学・コード生成において Pass@k 効率を改善し、創作でも多様性と整合性の両立を示した。
- 一言メモ: 直接の追跡テーマ外だが、複数候補生成から最良案を選ぶ金融分析エージェントの探索効率改善に転用余地がある。

### [T1][T4] Turning the TIDE: Cross-Architecture Distillation for Diffusion Large Language Models
- URL: https://arxiv.org/abs/2604.26951
- 重要度: 🔵Medium
- 要約:
  1. [課題] dLLM は並列復号の利点がある一方で大型化しやすく、異種アーキテクチャ間の蒸留手法が不足していた。
  2. [手法] 学習進行と diffusion timestep に応じた蒸留強度制御、重マスク時の補完文脈、cross-tokenizer 目的関数を組み合わせた TIDE を提案。
  3. [結果] 8B dense / 16B MoE 教師から 0.6B 学生への蒸留で8ベンチ平均1.53点改善し、HumanEval 48.78 を達成した。
- 一言メモ: MoE教師から小型学生へ知識移植する発想は、高性能基盤モデルを使って実運用向け軽量モデルを作る際の実装テンプレになる。

### [T1][T4] Accelerating RL Post-Training Rollouts via System-Integrated Speculative Decoding
- URL: https://arxiv.org/abs/2604.26779
- 重要度: ⚡High
- 要約:
  1. [課題] RL後学習ではロールアウト生成が律速になり、推論高速化が学習全体のボトルネックを左右する。
  2. [手法] speculative decoding を NeMo-RL + vLLM に統合し、同期・非同期の両パイプラインで lossless にロールアウト高速化する設計を示した。
  3. [結果] 8B reasoning workload の同期RLで1.8倍のロールアウトスループット、235B想定の非同期RLで最大2.5倍の学習高速化を見積もった。
- 一言メモ: PCIe環境で大型モデルを回すなら、量子化だけでなくロールアウト生成の speculative decoding 統合が学習コスト削減の本命になる。

### [T3] Recursive Multi-Agent Systems
- URL: https://arxiv.org/abs/2604.25917
- 重要度: ⚡High
- 要約:
  1. [課題] マルチエージェントは協調コストが高く、テキスト往復ベースの連携では推論速度とトークン効率が悪化しやすい。
  2. [手法] エージェント系全体を latent-space recursive computation として扱い、RecursiveLink による潜在状態共有と全体系の共同最適化を行う RecursiveMAS を提案。
  3. [結果] 9ベンチで平均8.3%の精度向上、1.2倍から2.4倍の推論高速化、34.6%から75.6%のトークン削減を達成した。
- 一言メモ: 金融AIエージェントの役割分担を増やすほど通信コストが痛むので、潜在状態共有型の協調設計は中期的にかなり重要。

### [T4] Why Fine-Tuning Encourages Hallucinations and How to Fix It
- URL: https://arxiv.org/abs/2604.15574
- 重要度: ⚡High
- 要約:
  1. [課題] SFTで新知識を学ばせると、事前学習知識との干渉で hallucination が増えることがある。
  2. [手法] 出力分布ドリフトを抑える self-distillation 型SFT と、必要に応じて factual plasticity を抑える parameter freezing を検証した。
  3. [結果] hallucination の主因が重なった意味表現間の干渉であることを示し、自己蒸留でその干渉を緩和できた。
- 一言メモ: 現在課題のハルシネーション抑制に直結し、金融SFTでは新知識注入より既存知識の破壊回避を先に設計すべきだと示す。

### [T2] Improving Robustness of Tabular Retrieval via Representational Stability
- URL: https://arxiv.org/abs/2604.24040
- 重要度: 🔵Medium
- 要約:
  1. [課題] 表構造を token sequence に潰す retrieval は、csv・tsv・html・markdown・ddl などの serialization 差で埋め込みがぶれやすい。
  2. [手法] 複数 serialization の埋め込みを共有意味信号の noisy view とみなし、centroid を canonical target にする表現安定化と adapter 学習を提案した。
  3. [結果] centroid 表現は複数 retriever で個別形式を上回る傾向を示し、軽量 adapter で dense retriever の頑健性も改善した。
- 一言メモ: XBRLや決算表のRAGでは、表の表現形式差を吸収する retrieval 設計が検索品質の底上げに効く。

### [T2] RaV-IDP: A Reconstruction-as-Validation Framework for Faithful Intelligent Document Processing
- URL: https://arxiv.org/abs/2604.23644
- 重要度: 🔵Medium
- 要約:
  1. [課題] IDP の抽出結果は原文忠実性を内在的に検証できず、誤抽出がそのままRAGや分析へ流れ込みやすい。
  2. [手法] 抽出後に再構成器で元の文書領域に近い形へ戻し、原画像との fidelity を比較して閾値未満なら vision fallback を発火する検証ループを導入した。
  3. [結果] reconstruction を第一級コンポーネントに据えた、ラベル不要の grounded quality signal を持つIDPパイプラインを提示した。
- 一言メモ: 決算書や社内PDFの構造化抽出で、抽出信頼度ではなく原文再構成一致で検証する発想は実装価値が高い。
