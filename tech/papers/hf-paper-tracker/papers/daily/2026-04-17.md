### [T1] Accelerating Speculative Decoding with Block Diffusion Draft Trees
- URL: https://arxiv.org/abs/2604.12989
- 重要度: ⚡High
- 要約:
  1. [課題] Speculative decoding は高速化に有効だが、既存の block diffusion drafter は1回の検証で1本のドラフト列しか扱えず受理長が伸びにくい。
  2. [手法] DDTree は block diffusion drafter の各位置分布から best-first に draft tree を構築し、ancestor-only attention mask で1回の target forward でまとめて検証する。
  3. [結果] DFlash 系を土台に受理効率をさらに改善し、leading class の speculative decoding 手法群に入る性能を示した。
- 一言メモ: PCIe 接続の多GPU MoE でレイテンシを削る打ち手として直結し、AWQ/INT4 前提の推論最適化候補として最優先で追う価値がある。

### [T2] MERRIN: A Benchmark for Multimodal Evidence Retrieval and Reasoning in Noisy Web Environments
- URL: https://arxiv.org/abs/2604.13418
- 重要度: 🔵Medium
- 要約:
  1. [課題] 現実の検索環境ではクエリが曖昧で情報源もノイジーかつマルチモーダルだが、既存ベンチはその難しさを十分に測れていない。
  2. [手法] MERRIN は動画・音声を含む複数モダリティの証拠検索と multi-hop reasoning を要求する human-annotated benchmark を構築した。
  3. [結果] 10モデルで評価しても平均精度は22.3%、最高でも40.1%に留まり、強い agent ほど過探索でノイズに引っ張られる傾向が示された。
- 一言メモ: 社内文書や市場情報を跨ぐ RAG でも「取りすぎて誤る」失敗はそのまま起きるので、検索段数と証拠選別の制御指標として参考になる。

### [T2] GlotOCR Bench: OCR Models Still Struggle Beyond a Handful of Unicode Scripts
- URL: https://arxiv.org/abs/2604.12978
- 重要度: 🔵Medium
- 要約:
  1. [課題] OCR 評価は一部の主要スクリプトに偏っており、多言語文書での実力が見えにくい。
  2. [手法] GlotOCR Bench は100超の Unicode script を対象に clean/degraded 画像を用意し、VLM 系 OCR の script 横断一般化を測る。
  3. [結果] 最強クラスのモデルでも30 script 未満しか十分に一般化できず、未知 script ではノイズ出力や近縁 script への hallucination が頻発した。
- 一言メモ: 多言語 IR・決算資料 OCR を扱うなら前処理の OCR 品質がボトルネックになり得るので、対応 script の事前監査が必要だと再確認できる。

### [T2/T4] BERT-as-a-Judge: A Robust Alternative to Lexical Methods for Efficient Reference-Based LLM Evaluation
- URL: https://arxiv.org/abs/2604.09497
- 重要度: 🔵Medium
- 要約:
  1. [課題] 生成評価で lexical match に依存すると、実力ではなく出力形式への追従度を測ってしまい、LLM judge は高コストになりやすい。
  2. [手法] 著者らは question-candidate-reference triplet から軽量 encoder を学習し、意味的正しさを効率的に判定する BERT-as-a-Judge を提案した。
  3. [結果] 36モデル・15タスクの検証で lexical baseline を一貫して上回り、大規模 LLM judge に匹敵する精度と低コストを両立した。
- 一言メモ: 構造化出力スキーマの評価でも、表層一致だけでなく意味整合を安価に回す評価器として転用余地がある。

### [T3] ClawGUI: A Unified Framework for Training, Evaluating, and Deploying GUI Agents
- URL: https://arxiv.org/abs/2604.11784
- 重要度: ⚡High
- 要約:
  1. [課題] GUI agent は RL 学習基盤、標準評価、実運用導線が分断されており、研究成果がそのまま実利用に繋がりにくい。
  2. [手法] ClawGUI は並列仮想環境と実機を扱う RL 基盤、6 benchmark を統一する評価系、複数 OS と chat platform への配備基盤を一つの harness に統合した。
  3. [結果] 公式 baseline の95.8%再現を達成し、ClawGUI-2B は MobileWorld GUI-Only で同規模 baseline を6.0ポイント上回る 17.1% success rate を記録した。
- 一言メモ: 学習・評価・配備を一気通貫で扱う設計は、金融エージェントでも MCP や社内ツール操作を統合する orchestration 層の参考になる。

### [T3] OccuBench: Evaluating AI Agents on Real-World Professional Tasks via Language World Models
- URL: https://arxiv.org/abs/2604.10866
- 重要度: ⚡High
- 要約:
  1. [課題] 実務エージェントの評価は公開環境が限られ、職種横断の能力差や障害耐性を測りにくい。
  2. [手法] OccuBench は Language World Models を使って100件の専門職シナリオを模擬し、完遂性能と fault injection 下の頑健性を同時評価する。
  3. [結果] モデルごとに得意業種が大きく分かれ、特に暗黙的データ欠損への耐性が弱く、推論 effort 増加で GPT-5.2 は 27.5 ポイント改善した。
- 一言メモ: 金融業務フロー向け agent 評価でも、単純正答率より「欠損・劣化データ下で壊れないか」を別軸で持つ必要がある。

### [T3] Memory Transfer Learning: How Memories are Transferred Across Domains in Coding Agents
- URL: https://arxiv.org/abs/2604.14004
- 重要度: ⚡High
- 要約:
  1. [課題] coding agent の memory は同一ドメイン内でしか活用されないことが多く、異種タスク間の再利用原理が不明だった。
  2. [手法] MTL は heterogeneous domain の共通 memory pool を作り、具体的 trace から抽象 insight まで4種の memory 表現で転移性を比較した。
  3. [結果] cross-domain memory は平均 3.7% 改善し、低レベル trace よりも validation routine などの抽象的 meta-knowledge が有効だと示した。
- 一言メモ: 金融エージェントでも個別案件の回答履歴より、検証手順や根拠確認フローを抽象 memory として蓄積する方が効きそうだ。

### [T3] Exploration and Exploitation Errors Are Measurable for Language Model Agents
- URL: https://arxiv.org/abs/2604.13151
- 重要度: ⚡High
- 要約:

  2. [手法] 部分観測グリッドと未知 DAG からなる controllable environment を設計し、行動列だけから exploration / exploitation error を測る metric を提案した。

- 一言メモ: RAG とツール利用を組み合わせる金融 agent では、検索し過ぎと既知根拠の使い切れなさを別々に可視化する運用指標として有用。

### [T3] TREX: Automating LLM Fine-tuning via Agent-Driven Tree-based Exploration
- URL: https://arxiv.org/abs/2604.14116

- 要約:

  2. [手法] TREX は Researcher と Executor の二役を中心に、探索過程を search tree として再利用しながら LLM fine-tuning の全工程を回す multi-agent system を構築した。

### [T3] SkVM: Compiling Skills for Efficient Execution Everywhere
- URL: https://arxiv.org/abs/2604.03088
- 重要度: 🔵Medium
- 要約:
  1. [課題] skill を単なる prompt context として共有すると、モデルや harness が変わるだけで挙動が崩れやすい。
  2. [手法] SkVM は skill を code と見なし、primitive capability 分解に基づく compile-time 最適化と runtime の JIT solidification / adaptive recompilation を行う。
  3. [結果] 8モデル・3 harness で task completion を改善しつつ、token 消費を最大40%削減、並列性強化で最大3.2倍高速化した。
- 一言メモ: 社内 agent skill をモデル非依存で移植したいなら、prompt 断片ではなく capability-aware な中間表現を持つ発想が効く。

### [T3] The Blind Spot of Agent Safety: How Benign User Instructions Expose Critical Vulnerabilities in Computer-Use Agents
- URL: https://arxiv.org/abs/2604.10577
- 重要度: ⚡High
- 要約:
  1. [課題] computer-use agent の安全評価は明示的な悪意指示に偏っており、 benign instruction でも実行文脈で有害化するケースを捉えられていない。
  2. [手法] OS-BLIND は environment-embedded threats と agent-initiated harms を含む 300 タスクの benchmark を作り、単体 agent と multi-agent system を比較評価した。
  3. [結果] 多くの CUA が 90%超の attack success rate を示し、安全調整済みモデルでも multi-agent 化で 73.0% から 92.7% へ悪化した。
- 一言メモ: benign な業務依頼でも市場操作や誤発注に繋がる金融ドメインでは、最初の safety check だけでなく実行中の再評価が必要だと示唆する。

### [T4] Self-Distillation Zero: Self-Revision Turns Binary Rewards into Dense Supervision
- URL: https://arxiv.org/abs/2604.12002
- 重要度: ⚡High
- 要約:
  1. [課題] RLVR は binary reward が疎で学習効率が低く、dense supervision を得るには外部教師や高品質デモが必要になりやすい。
  2. [手法] SD-Zero は同一モデルに Generator と Reviser の二役を持たせ、reward 付き応答から reviser の token 分布を generator へ on-policy distillation する。
  3. [結果] Qwen3-4B と Olmo-3-7B で base model から 10%以上改善し、同一予算で RFT・GRPO・SDFT を上回った。
- 一言メモ: 正誤判定しやすい構造化出力タスクなら、外部教師なしで schema 遵守を詰める post-training レシピとしてかなり相性がよい。

### [T4] Target Policy Optimization
- URL: https://arxiv.org/abs/2604.06159
- 重要度: ⚡High
- 要約:
  1. [課題] PPO や GRPO などの policy gradient 系は「何を増やすか」と「どう更新するか」を同時に処理するため、疎報酬下で更新量が不安定になりやすい。
  2. [手法] TPO は scored completion から target distribution を先に作り、その分布へ cross-entropy で policy を合わせる二段分離型の最適化を行う。
  3. [結果] bandit、sequence task、billion-parameter LLM RLVR で容易課題は既存法と同等、疎報酬課題では明確に上回った。
- 一言メモ: 構造化出力の pass/fail 報酬は疎くなりがちなので、DPO/KTO の先に置く RLVR 候補として試す価値が高い。

### [T4] TIP: Token Importance in On-Policy Distillation
- URL: https://arxiv.org/abs/2604.14084
- 重要度: ⚡High
- 要約:
  1. [課題] on-policy distillation では全 token を同じ重みで学習しがちで、GPU メモリと学習効率の無駄が大きい。
  2. [手法] TIP は token 重要度を student entropy と teacher-student divergence の2軸で整理し、不確実 token と「自信過剰だが誤り」の token を重点抽出する。
  3. [結果] entropy ベースで 50% token 学習でも全 token 学習並み以上を達成し、重要領域だけなら 10%未満でも近い性能を出しつつ peak memory を最大47%削減した。
- 一言メモ: RTX 6000 Pro 環境で大規模 teacher から distill する際、限られた VRAM で学習効率を上げる具体策になる。

### [T4] From P(y|x) to P(y): Investigating Reinforcement Learning in Pre-train Space
- URL: https://arxiv.org/abs/2604.14142
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLVR は条件付き分布 P(y|x) の改善に閉じるため、ベースモデルの探索空間そのものが弱いと伸びしろが制限される。
  2. [手法] PreRL は事前学習空間の周辺分布 P(y) に online reward 更新をかけ、Negative Sample Reinforcement で誤った reasoning 空間を先に削る DSRL を提案した。
  3. [結果] log P(y) と log P(y|x) の勾配整合を示しつつ、NSR-PreRL により reflection thought を増やし、強い baseline 群を一貫して上回った。
- 一言メモ: 金融推論で hallucination を減らすには、応答条件付け後の調整だけでなくベースの出力空間そのものを狭める発想が効くかもしれない。
