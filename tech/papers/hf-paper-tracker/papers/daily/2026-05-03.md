### [T1] Efficient Training on Multiple Consumer GPUs with RoundPipe
- URL: https://arxiv.org/abs/2604.27085
- 重要度: ⚡High
- 要約:
  1. [課題] 消費者向けGPUのLoRA微調整は、PCIe帯域の遅さとステージ偏りによる pipeline bubble でスループットが頭打ちになる。
  2. [手法] RoundPipe はGPUを固定ステージ担当ではなく stateless worker として扱い、round-robin dispatch と優先度付き転送制御で near-zero-bubble のPPを実現する。
  3. [結果] 8x RTX 4090で既存法比1.48-2.16倍高速化し、Qwen3-235Bの31K長文LoRA微調整まで単一サーバで可能にした。
- 一言メモ: PCIe前提の自社環境に直結しており、MoE系のSFT/QLoRA基盤を現実的に押し上げる即試験候補。

### [T1] Nemotron 3 Nano Omni: Efficient and Open Multimodal Intelligence
- URL: https://arxiv.org/abs/2604.24954
- 重要度: 🔵Medium
- 要約:
  1. [課題] マルチモーダル基盤モデルは性能と推論効率の両立が難しく、文書理解やagentic computer useでは特に遅延が問題になる。
  2. [手法] 30B-A3Bの高効率バックボーン上に音声対応と multimodal token reduction を組み合わせ、BF16/FP8/FP4 形式で公開した。
  3. [結果] 先行Nemotron比で全モダリティの精度を改善しつつ、同規模帯でより低遅延・高スループットを達成した。
- 一言メモ: A3B疎構造と低精度配布は、将来の金融ドキュメント理解やGUI操作エージェントの実装選定に参考になる。

### [T1] Turning the TIDE: Cross-Architecture Distillation for Diffusion Large Language Models
- URL: https://arxiv.org/abs/2604.26951
- 重要度: 🔵Medium
- 要約:
  1. [課題] dLLMは並列復号の魅力がある一方で、性能確保に大規模化が必要で、異種アーキテクチャ間の知識蒸留手法も未整備だった。
  2. [手法] TIDE は timestep依存の蒸留強度制御、補完的mask split、cross-tokenizer目的を組み合わせて異種教師から学生へ蒸留する。
  3. [結果] 8B dense と16B MoE教師から0.6B学生を蒸留し、8ベンチ平均で既存法を上回り、HumanEvalもAR基線を大きく超えた。
- 一言メモ: 大型MoEの能力を軽量補助モデルへ落とす設計として、社内の下流ツール用サブモデル作成に転用余地がある。

### [T1, T4] Accelerating RL Post-Training Rollouts via System-Integrated Speculative Decoding
- URL: https://arxiv.org/abs/2604.26779
- 重要度: ⚡High
- 要約:
  1. [課題] RL後学習ではロールアウト生成がボトルネックで、特に大規模モデルでは学習全体の速度を強く制約する。
  2. [手法] NeMo-RL と vLLM に speculative decoding を統合し、同期・非同期RLの両方で損失なしに rollout を高速化する実装を示した。
  3. [結果] 8B同期RLで1.8倍の rollout throughput を達成し、235B級では非同期化と組み合わせて最大2.5倍の end-to-end 学習短縮を予測した。
- 一言メモ: DPO/KTO以外にRL系アライメントへ踏み込む場合、推論高速化をそのまま学習速度に変換できる点が実務的に大きい。

### [T2, T3] Operating-Layer Controls for Onchain Language-Model Agents Under Real Capital
- URL: https://arxiv.org/abs/2604.26091
- 重要度: ⚡High
- 要約:
  1. [課題] 実資金を扱う金融エージェントでは、基盤モデル単体の性能よりも、誤ったルール生成や数値誤読を防ぐ運用層の設計が成否を左右する。
  2. [手法] 実運用されたオンチェーン売買エージェントの大規模ログを基に、prompt compilation、typed controls、policy validation、execution guards、memory設計を分析した。
  3. [結果] 21日で7.5M invocations、約300K onchain actions、約$20M出来高を処理しつつ、特定のハーネス改善で fabricated sell rules を57%から3%まで低減した。
- 一言メモ: 構造化出力スキーマ品質を最重要課題に置く現プロジェクトに対し、「モデル改善より先に運用層で縛る」方針を強く裏づける。

### [T3] Step-level Optimization for Efficient Computer-use Agents
- URL: https://arxiv.org/abs/2604.27151
- 重要度: ⚡High
- 要約:
  1. [課題] computer-use agent は各ステップで常に大型MMモデルを呼ぶため、長手数タスクほどコストと遅延が膨らむ。
  2. [手法] 小型ポリシーを通常時に使い、Stuck Monitor と Milestone Monitor が高リスク時だけ強いモデルへエスカレーションする step-level cascade を提案した。
  3. [結果] 既存エージェントへ後付けできる構成で、進捗停滞と silent semantic drift という代表的失敗を計算配分で抑える道筋を示した。
- 一言メモ: 金融業務フローでも「全ステップ最強モデル」は非現実的なので、監視器付き段階推論はすぐ設計に取り込める。

### [T3] Claw-Eval-Live: A Live Agent Benchmark for Evolving Real-World Workflows
- URL: https://arxiv.org/abs/2604.28139
- 重要度: ⚡High
- 要約:
  1. [課題] 多くのエージェント評価は固定タスクと最終回答採点に偏り、実運用で必要な行動検証や需要変化への追随が弱い。
  2. [手法] 公開ワークフロー需要から刷新される signal layer と、再現可能な release snapshot を分離した live benchmark を構築した。
  3. [結果] 105タスク・13モデル評価で首位でも66.7%しか通過せず、HRや複数システム横断業務が持続的な難所だと分かった。
- 一言メモ: 自社評価も最終回答中心では不足で、実行痕跡・監査ログ・成果物まで見る設計へ寄せる必要がある。

### [T3, T4] ClawGym: A Scalable Framework for Building Effective Claw Agents
- URL: https://arxiv.org/abs/2604.26904
- 重要度: ⚡High
- 要約:
  1. [課題] ローカルファイルやツールをまたぐ Claw 型エージェントは、検証可能な訓練データと診断評価の整備が不足している。
  2. [手法] persona駆動で13.5K件の合成タスクを生成し、その black-box rollout でSFTしつつ、task sandbox 並列化による軽量RLも試した。
  3. [結果] SynData、Agents、Bench の一体化フレームワークを示し、個人業務エージェント開発の全ライフサイクルをスケール可能にした。
- 一言メモ: 社内文書RAGやMCP連携を含む業務エージェントでは、同様の合成タスク生成からSFTデータを作る流れが有効。

### [T3] AutoResearchBench: Benchmarking AI Agents on Complex Scientific Literature Discovery
- URL: https://arxiv.org/abs/2604.25256
- 重要度: 🔵Medium
- 要約:
  1. [課題] 論文探索のような open-ended research task は既存の web-browsing ベンチでは十分に測れない。
  2. [手法] 特定論文へ辿り着く Deep Research と条件を満たす集合を集める Wide Research の2種からなる専用ベンチを作成した。
  3. [結果] 強力なLLMでも Deep Research 正答率9.39%、Wide Research IoU 9.31%と極めて低く、研究探索エージェントの未成熟さが明確になった。
- 一言メモ: 論文追跡や企業調査エージェントを作るなら、検索能力を過信せず、探索戦略そのものを別途鍛える必要がある。

### [T3] Heterogeneous Scientific Foundation Model Collaboration
- URL: https://arxiv.org/abs/2604.27351
- 重要度: 🔵Medium
- 要約:
  1. [課題] LLM中心のエージェントは、構造化データや専門モダリティを扱うドメイン基盤モデルを十分に活かせない。
  2. [手法] Eywa は専門基盤モデルに言語モデル由来の reasoning interface を被せ、planner が通常エージェントと専門エージェントを動的協調させる。
  3. [結果] 物理・生命・社会科学の複数領域で、言語偏重を減らしつつ構造化データ系タスクの性能を改善した。
- 一言メモ: 金融でも価格時系列・表形式・XBRL専用モデルを LLM オーケストレータの配下に置く設計が有望だと示す。

### [T4] Co-Evolving Policy Distillation
- URL: https://arxiv.org/abs/2604.27083
- 重要度: ⚡High
- 要約:
  1. [課題] 複数能力を1モデルへ統合する際、mixed RLVR は能力干渉が強く、後段の一括蒸留も教師と学生の行動差で取りこぼしが出る。
  2. [手法] CoPD は各 expert の RLVR 学習中から双方向OPDを並行実施し、expert 同士を相互教師として co-evolve させる。
  3. [結果] テキスト・画像・動画 reasoning の all-in-one 統合で既存法を上回り、ドメイン別 expert すら超える性能を示した。
- 一言メモ: 金融、RAG、構造化出力など別能力を個別最適してから統合したい現状に対し、能力干渉を減らす学習レシピ候補になる。

### [T4] Safety Drift After Fine-Tuning: Evidence from High-Stakes Domains
- URL: https://arxiv.org/abs/2604.24902
- 重要度: 🔵Medium
- 要約:
  1. [課題] 多くの運用では base model の安全性評価をそのまま fine-tune 後にも引き継げると暗黙に仮定している。
  2. [手法] 医療・法務を含む100モデルについて、base と fine-tune 後の安全性を複数ベンチで比較した。
  3. [結果] 無害に見える fine-tuning でも安全性指標は大きく揺れ、ある評価では改善し別評価では悪化する不安定性が確認された。
- 一言メモ: 金融特化SFTやDPOを回した後は、ベース評価の再利用ではなく、実運用文脈で再評価する工程を必須化すべき。
