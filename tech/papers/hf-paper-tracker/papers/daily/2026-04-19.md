### [T4] Reinforcement Learning via Value Gradient Flow
- URL: https://arxiv.org/abs/2604.14265
- 重要度: 🔵Medium
- 要約:
  1. [課題] 行動分布から外れた領域で価値を過大最適化してしまう既存RLは、LLM後学習やオフラインRLで安定にスケールしにくい。
  2. [手法] 参照分布から価値誘導分布への写像を最適輸送として定式化し、粒子を価値勾配で動かす離散的 gradient flow により暗黙的な正則化を実現する VGF を提案した。
  3. [結果] D4RL・OGBench・LLM RL タスクで既存法を上回り、輸送予算の調整で test-time scaling も可能にした。
- 一言メモ: DPO/KTO 系の代替ではないが、報酬最適化を安定化する設計として金融特化後学習の暴走抑制に応用余地がある。

### [T2][T3] Don't Retrieve, Navigate: Distilling Enterprise Knowledge into Navigable Agent Skills for QA and RAG
- URL: https://arxiv.org/abs/2604.14572
- 重要度: ⚡High
- 要約:
  1. [課題] 通常のRAGは検索結果の受け身消費に留まり、コーパス構造を見渡せないため探索の戻りや証拠統合が弱い。
  2. [手法] 文書群を階層的スキルディレクトリへオフライン蒸留し、推論時はエージェントが要約ツリーを段階的に辿って本文取得する Corpus2Skill を提案した。
  3. [結果] 企業向け QA ベンチマーク WixQA で dense retrieval、RAPTOR、agentic RAG を全評価指標で上回った。
- 一言メモ: 社内文書・XBRL・決算資料のRAGを「検索」から「ナビゲーション」へ変える発想で、根拠追跡性と再探索性の改善に直結する。

### [T4] Switch-KD: Visual-Switch Knowledge Distillation for Vision-Language Models
- URL: https://arxiv.org/abs/2604.14629
- 重要度: 🔵Medium
- 要約:
  1. [課題] VLM 蒸留では視覚側とテキスト側を別々に教師信号化しがちで、マルチモーダル整合が崩れて知識移転が不安定になる。
  2. [手法] 学生の視覚出力を教師の言語経路に通して共通のテキスト確率空間で蒸留する Visual-Switch Distillation と、双方向で確率差分を揃える DBiLD 損失を導入した。
  3. [結果] 0.5B TinyLLaVA が 3B 教師から知識を受け継ぎ、10ベンチ平均で 3.6 ポイント改善した。
- 一言メモ: 財務資料VLMを小型化して運用する際、モダリティ間整合を保った蒸留レシピとして参考になる。

### [T3] Towards Autonomous Mechanistic Reasoning in Virtual Cells
- URL: https://arxiv.org/abs/2604.11661
- 重要度: 🔵Medium
- 要約:
  1. [課題] 科学発見向けエージェントは、事実に裏づけられた検証可能な説明を作れず、オープンエンド研究で信頼しにくい。
  2. [手法] 生物学的推論を mechanistic action graph で表し、知識検索と検証器を組み合わせるマルチエージェント VCR-Agent と検証済み説明データ VC-TRACES を構築した。
  3. [結果] 検証付き説明で学習すると factual precision と下流の遺伝子発現予測が改善し、研究エージェントの信頼性向上を示した。
- 一言メモ: 金融リサーチでも「結論」ではなく「検証可能な推論グラフ」を残す設計が、監査性とハルシネーション抑制の鍵になる。

### [T4] How to Fine-Tune a Reasoning Model? A Teacher-Student Cooperation Framework to Synthesize Student-Consistent SFT Data
- URL: https://arxiv.org/abs/2604.14164
- 重要度: ⚡High
- 要約:
  1. [課題] 強い教師が生成した合成データで reasoning model を SFT すると、教師と生徒の文体分布のズレで性能が落ちることがある。
  2. [手法] 教師と生徒が交互にトークン生成し、教師の推論力と生徒の文体整合を両立する合成データを作る TESSY を提案した。
  3. [結果] GPT-OSS-120B 教師で生成した通常データでは Qwen3-8B が LiveCodeBench-Pro で 3.25% 低下した一方、TESSY は 11.25% 改善し、OJBench でも 6.68% 改善した。
- 一言メモ: 金融SFTでも「良い答え」だけでなく「自モデルに馴染む書き方」でデータを作る重要性を示している。

### [T4] C2: Scalable Rubric-Augmented Reward Modeling from Binary Preferences
- URL: https://arxiv.org/abs/2604.13618
- 重要度: 🔵Medium
- 要約:
  1. [課題] rubric 付き reward model は判断の信頼性が高いが、良質な rubric 注釈の作成コストが高く、低品質 rubric はむしろ有害になる。
  2. [手法] 二値選好だけから「役立つ rubric / 誤誘導 rubric」の対を合成し、協調的 rubric generator と批判的 verifier を同時学習する C2 を提案した。
  3. [結果] 外部 rubric 注釈なしで RM-Bench で最大 6.5 ポイント、AlpacaEval 2.0 長さ制御 win rate で 6.0 ポイント改善し、4倍大きいモデル由来 rubric に匹敵した。
- 一言メモ: 構造化出力の評価軸を rubric 化し、少量の選好から reward model を育てる方向にそのまま繋がる。

### [T4] LongAct: Harnessing Intrinsic Activation Patterns for Long-Context Reinforcement Learning
- URL: https://arxiv.org/abs/2604.14922
- 重要度: 🔵Medium
- 要約:
  1. [課題] 長文脈RLは全重みを一様更新しがちで、実際に重要な長距離依存の特徴へ学習が集中しにくい。
  2. [手法] 長文脈処理時の高振幅 Q/K 活性に着目し、その活性に結び付く重みだけを疎に更新する saliency-guided sparse update を導入した。
  3. [結果] LongBench v2 で約 8% 改善し、RULER でも汎化を伸ばし、GRPO や DAPO など複数RL法で一貫して有効だった。
- 一言メモ: 長い決算資料や社内文書を扱う金融エージェントの長文脈後学習を、低コストで効率化するヒントになる。

### [T2][T3] UniDoc-RL: Coarse-to-Fine Visual RAG with Hierarchical Actions and Dense Rewards
- URL: https://arxiv.org/abs/2604.14967
- 重要度: 🔵Medium
- 要約:
  1. [課題] Visual RAG は粗い検索信号に依存し、複雑な文書推論で必要な細粒度の視覚証拠を取りこぼしやすい。
  2. [手法] 文書検索、再ランキング、画像選択、領域クロップ、推論を階層アクションとして統合し、各段階に dense reward を与える RL フレームワーク UniDoc-RL を提案した。
  3. [結果] 3つのベンチマークで先行RL系 Visual RAG を最大 17.7% 上回った。
- 一言メモ: 決算PDFや図表付き資料のRAGで、ページ→図表→領域へと掘り下げる金融ドキュメント読解エージェントに近い。

### [T1] KV Packet: Recomputation-Free Context-Independent KV Caching for LLMs
- URL: https://arxiv.org/abs/2604.13226
- 重要度: ⚡High
- 要約:
  1. [課題] 既存の KV cache 再利用は文脈依存で、別コンテキストに差し込む際に再計算が必要となり TTFT を押し上げる。
  2. [手法] キャッシュ済み文書を不変パケットとして扱い、軽量 soft-token adapter を自己教師あり蒸留で学習して文脈の継ぎ目だけを補正する KV Packet を提案した。
  3. [結果] Llama-3.1 と Qwen2.5 で再計算ベースラインに近い F1 を保ちつつ、ほぼゼロ FLOPs でより低い TTFT を実現した。
- 一言メモ: 社内RAGの頻出資料や定番プロンプト断片を再利用しやすく、PCIe 環境での待ち時間削減に直結する。

### [T2] ROSE: An Intent-Centered Evaluation Metric for NL2SQL
- URL: https://arxiv.org/abs/2604.12988
- 重要度: ⚡High
- 要約:
  1. [課題] NL2SQL の標準指標 Execution Accuracy は構文差や誤った正解SQLに弱く、ユーザー意図との整合を測れていない。
  2. [手法] 予測SQLが質問意図に沿うかを独立評価する Prover と、正解SQLを証拠に反証する Refuter を組み合わせた intent-centered metric の ROSE を提案した。
  3. [結果] 専門家評価との一致で次点手法を Cohen's Kappa で約 24% 上回り、19手法の再評価から既存ランキングの歪みも示した。
- 一言メモ: 構造化出力スキーマ品質が最重要課題の現状では、SQL に限らず「意図中心評価」へ寄せる発想がそのまま効く。

### [T4] Self-Distillation Zero: Self-Revision Turns Binary Rewards into Dense Supervision
- URL: https://arxiv.org/abs/2604.12002
- 重要度: ⚡High
- 要約:
  1. [課題] RLVR は二値報酬しか得られず学習信号が疎で、外部教師の高品質デモに頼る蒸留はコストが高い。
  2. [手法] 同一モデルに Generator と Reviser の二役を担わせ、回答と二値報酬から改善版を生成させて、そのトークン分布を元回答側へ自己蒸留する SD-Zero を提案した。
  3. [結果] Qwen3-4B-Instruct と Olmo-3-7B-Instruct の数理・コード推論でベースモデル比 10%以上改善し、RFT・GRPO・SDFT を同条件で上回った。
- 一言メモ: 人手ラベルを増やしにくい金融タスクでも、検証可能な正誤信号から密な教師信号を自作できるのが強い。

### [T3] SkVM: Compiling Skills for Efficient Execution Everywhere
- URL: https://arxiv.org/abs/2604.03088
- 重要度: 🔵Medium
- 要約:
  1. [課題] スキルをただの長文コンテキストとして扱う現行エージェントは、モデルやハーネスが変わると移植性と効率が急落する。
  2. [手法] 11.8万件のスキル分析を基に、必要能力を分解して capability-based compilation、環境束縛、並列性抽出、JIT 固化を行う SkVM を提案した。
  3. [結果] 複数モデル・3種ハーネスで成功率を改善し、トークン消費を最大 40% 削減、並列化で 3.2 倍高速化、code solidification で 19〜50 倍の遅延削減を達成した。
- 一言メモ: MCP や社内スキル群をまたぐ金融エージェントで、スキルを資産化して再利用性能を上げる設計として重要。

### [T4] Target Policy Optimization
- URL: https://arxiv.org/abs/2604.06159
- 重要度: ⚡High
- 要約:
  1. [課題] PPO/GRPO 系は「どの出力に確率質量を移すか」と「どう更新するか」を同時に解くため、学習率や clipping に過敏で疎報酬で不安定になりやすい。
  2. [手法] スコア済み候補から目標分布 q を先に作り、その分布へ cross-entropy で追従する二段階最適化 TPO を提案した。
  3. [結果] bandit、Transformer 系列タスク、billion-scale LLM RLVR で easy task は既存法並み、疎報酬では PG・PPO・GRPO・DG を大きく上回った。
- 一言メモ: スキーマ遵守や拒否判断のような sparse reward 設定で、後学習の安定化候補としてかなり実務的。

### [T3] Memory Transfer Learning: How Memories are Transferred Across Domains in Coding Agents
- URL: https://arxiv.org/abs/2604.14004
- 重要度: ⚡High
- 要約:
  1. [課題] エージェントの記憶活用は同一ドメイン内に閉じがちで、異種タスク間の共通知見をうまく再利用できていない。
  2. [手法] 異なるベンチマークを横断する統一メモリプールを作り、具体的な実行痕跡から抽象的インサイトまで4種の記憶表現で転移性を比較した。
  3. [結果] 6つのコーディングベンチで平均 3.7% 改善し、低レベル痕跡より高レベルな検証ルーチンやメタ知識が転移を主導することを示した。
- 一言メモ: 金融エージェントでも「この案件固有の回答」より「検証手順や失敗パターン」を記憶化する方が再利用価値が高い。

### [T3] Exploration and Exploitation Errors Are Measurable for Language Model Agents
- URL: https://arxiv.org/abs/2604.13151
- 重要度: 🔵Medium
- 要約:
  1. [課題] LM agent の失敗を「探索不足」と「活用不足」に分解して測る指標がなく、改善ポイントが見えにくい。
  2. [手法] 部分観測グリッドと未知DAGからなる可制御環境を作り、行動列から exploration / exploitation error を policy-agnostic に算出する評価法を設計した。
  3. [結果] 最先端エージェントでも両面の失敗が顕著で、reasoning model と最小限の harness engineering が大きく改善することを示した。
- 一言メモ: マルチエージェントの計画失敗を「探索」「活用」に分けて観測できれば、金融ワークフローのボトルネック分析がやりやすくなる。

### [T4] From P(y|x) to P(y): Investigating Reinforcement Learning in Pre-train Space
- URL: https://arxiv.org/abs/2604.14142
- 重要度: 🔵Medium
- 要約:
  1. [課題] RLVR は条件付き分布 P(y|x) の調整に留まり、ベースモデルが元々持つ出力空間そのものの制約を越えにくい。
  2. [手法] 事前学習空間の周辺分布 P(y) を報酬駆動で更新する PreRL と、負例強化で誤答空間を削ってから通常RLへ移る DSRL を提案した。
  3. [結果] log P(y) と log P(y|x) の勾配整合を示しつつ、NSR-PreRL により transition thought を 14.89 倍、reflection thought を 6.54 倍増やし、強いベースラインを上回った。
- 一言メモ: 金融推論の土台能力そのものを押し上げたいなら、SFT後の微修正だけでなく pre-train space 側の介入も視野に入る。
